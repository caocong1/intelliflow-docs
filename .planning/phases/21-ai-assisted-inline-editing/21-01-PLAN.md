  <name>Task 1: Create inline-edit service with prompt construction, security check, and audit logging</name>
  <files>packages/backend/src/modules/runtime/inline-edit.service.ts</files>
  <action>
Create `inline-edit.service.ts` with the following functions:

1. **`buildInlineEditPrompt(action, selectedText, customInstruction?)`** — Returns a prompt string based on the action type:
   - `rewrite`: "请改写以下文本，保持原意但使用不同的表达方式：\n\n{text}\n\n请只返回修改后的文本，不要添加解释或额外内容。"
   - `simplify`: "请精简以下文本，去除冗余，保持核心信息：\n\n{text}\n\n请只返回修改后的文本，不要添加解释或额外内容。"
   - `expand`: "请扩写以下文本，增加细节和深度，保持原有风格：\n\n{text}\n\n请只返回修改后的文本，不要添加解释或额外内容。"
   - `fix`: "请修正以下文本中的语法、拼写和标点错误，保持原意：\n\n{text}\n\n请只返回修改后的文本，不要添加解释或额外内容。"
   - `custom`: "{customInstruction}\n\n{text}\n\n请只返回修改后的文本，不要添加解释或额外内容。"
   - Throw error for unknown action.

2. **`isPostRestoreNode(documentId, nodeExecutionId)`** — Query `nodeExecutions` table for the document, ordered by stepOrder. Find the target nodeExecutionId's stepOrder. Check if any preceding node with `nodeType = 'restore'` and `status = 'completed'` exists. Return boolean.

3. **`validateModelSecurity(modelId, isPostRestore)`** — If `isPostRestore` is true, look up the model's provider `deploymentType` via a join query. If `deploymentType !== 'local'`, throw AppError.forbidden("安全约束：恢复节点后仅允许使用本地模型"). If not post-restore, allow any model.

4. **`executeInlineEdit(documentId, nodeExecutionId, modelId, prompt, userId)`** —
   - Look up the model record to get providerId, then get the provider record to get strategy config (baseUrl, apiKey, modelName).
   - Use the same strategy pattern as model-call.service.ts: get the appropriate strategy (openai-compatible or claude-agent-sdk) based on provider type.
   - Create a ReadableStream that:
     a. Calls the strategy's `executeStream()` with a single-model setup
     b. Emits SSE events: `data: {"type":"delta","modelId":"...","data":"..."}`, `data: {"type":"complete","modelId":"...","data":"..."}`, `data: {"type":"error","modelId":"...","data":"..."}`
     c. After completion, inserts a record into `model_call_logs` with `callSource: 'inline_edit'`, recording tokenUsage if available
   - Return the ReadableStream.

Follow the exact SSE event format used in model-call.service.ts. Import strategy resolution from the existing strategies. Use the existing `db` instance and schema imports.

**inline-edit.routes.ts:**

Create a new Elysia route group with prefix `/runtime` using `requireAuth` guard.

Define a single POST endpoint: `/:documentId/inline-edit/:nodeExecutionId/stream`

Request body schema (use Elysia `t` schema):
```typescript
body: t.Object({
  action: t.Union([
    t.Literal("rewrite"),
    t.Literal("simplify"),
    t.Literal("expand"),
    t.Literal("fix"),
    t.Literal("custom"),
  ]),
  selectedText: t.String({ minLength: 1 }),
  modelId: t.String({ minLength: 1 }),
  customInstruction: t.Optional(t.String()),
})
```

Endpoint logic:
1. Verify user is a project member for the document (use `isDocumentProjectMember` from versions.service, same pattern as model-call.routes.ts)
2. Call `isPostRestoreNode(documentId, nodeExecutionId)` to determine security context
3. Call `validateModelSecurity(modelId, isPostRestore)` — will throw 403 if violation
4. Call `buildInlineEditPrompt(action, selectedText, customInstruction)`
5. Call `executeInlineEdit(documentId, nodeExecutionId, modelId, prompt, userId)`
6. Return `new Response(stream, { headers: { "Content-Type": "text/event-stream", "Cache-Control": "no-cache", Connection: "keep-alive" } })`

Use POST (not GET) because selectedText can be large and would exceed URL length limits.

**runtime.routes.ts:**

Import `inlineEditRoutes` and mount it alongside the existing route groups using `.use(inlineEditRoutes)`.
