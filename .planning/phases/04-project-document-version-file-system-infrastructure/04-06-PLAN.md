<!-- Root cause analysis -->

The TS2322 errors occur at WorkflowEditor.tsx lines 166-167 where `backendNodes` and `backendEdges` are passed to `.put()`. Eden Treaty infers the body type from the backend route schema. In workflows.routes.ts:

```typescript
const nodeSchema = t.Any();  // <-- This causes Eden to infer File | File[]
const edgeSchema = t.Any();  // <-- Same problem

// PUT :id body uses these:
body: t.Object({
  name: t.Optional(t.String({ minLength: 1, maxLength: 200 })),
  description: t.Optional(t.Nullable(t.String({ maxLength: 1000 }))),
  nodes: t.Optional(t.Array(nodeSchema)),   // inferred as File[]
  edges: t.Optional(t.Array(edgeSchema)),   // inferred as File[]
})
```

The fix: Replace `t.Any()` with a proper object schema that matches WorkflowNodeDef and WorkflowEdgeDef, or use `t.Unsafe<unknown>()` / structured `t.Object()` definitions that Eden can correctly infer as non-File types.

From packages/shared/src/types.ts:
```typescript
export interface WorkflowNodeDef {
  id: string;
  type: WorkflowNodeType;
  label: string;
  position: { x: number; y: number };
  config: Record<string, unknown>;
  outputs: Array<{ name: string; label: string }>;
}

export interface WorkflowEdgeDef {
  id: string;
  source: string;
  target: string;
  sourceHandle?: string;
  targetHandle?: string;
}
```

Additionally, there are already uncommitted changes in git working tree that partially fix WorkflowEditor.tsx (removing `as unknown as` casts for API calls) and WorkflowCanvas.tsx (simplifying CanvasInner). These changes should be incorporated.
