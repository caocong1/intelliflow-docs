**A) Create `notifications.service.ts`**:

1. `createNotification(params: { userId: string; type: "generation_completed" | "generation_failed"; title: string; message?: string; documentId?: string; projectId?: string })`:
   - Insert into notifications table, return the created notification

2. `getNotifications(userId: string, opts?: { limit?: number; offset?: number })`:
   - Select from notifications where userId matches, ordered by createdAt desc
   - Default limit 50, max 100
   - Return { notifications, total } for pagination

3. `getUnreadCount(userId: string)`:
   - Count notifications where userId matches and isRead = false
   - Return { count: number }

4. `markRead(notificationId: string, userId: string)`:
   - Update isRead = true where id matches AND userId matches (security: only own notifications)

5. `markAllRead(userId: string)`:
   - Update all notifications where userId matches and isRead = false, set isRead = true

**B) Create `notifications.routes.ts`**:

Create Elysia router with prefix "/notifications", using the same auth guard pattern as other routes:

1. `GET /notifications` — calls getNotifications, accepts query params: limit, offset
2. `GET /notifications/unread-count` — calls getUnreadCount, returns { count }
3. `PATCH /notifications/:id/read` — calls markRead
4. `PATCH /notifications/read-all` — calls markAllRead

**C) Register routes in `index.ts`**:
   - Import notificationRoutes
   - Add `.use(notificationRoutes)` to the Elysia app chain (same pattern as existing route registrations)
