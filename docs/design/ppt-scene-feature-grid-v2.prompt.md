# PPT Scene Single-Slide Prompt: Feature Grid V2

Model:

`gemini-3.1-pro-preview`

Goal:

生成一页高质量、高设计感的特性拼贴页，输出完整 `ppt_scene/v1` JSON，但 `slides` 数组中只能有 1 页。

Prompt:

```text
基于已提供的 `ppt_scene/v1` 协议，输出一个完全合法的 JSON。

任务：
- 只生成 1 页
- 页类型：feature grid
- 主题：无线网络建设全流程指南
- 语言：zh-CN

内容主题：
主流无线 AP 设备类型及特点

内容点：
- 面板 AP：美观，适合独立房间
- 放装 AP：覆盖范围广，适合开放区域
- 高密度 AP：适合会议、会场、教学场景
- 室外 AP：适合园区、广场、停车场

质量目标：
- 必须像“设备图谱海报 / 特性拼贴页”
- 不能像普通 2x2 或 1大3小卡片页
- 必须更像 editorial board，而不是功能网格

视觉方向：
- editorial atlas
- collage
- premium
- product language
- modular but asymmetric

强制要求：
1. 只输出一个完整的 `ppt_scene/v1` JSON。
2. `slides` 中必须且只能有 1 页。
3. 禁止四个重复信息卡。
4. 必须有一个主模块，至少占页面 35% 以上视觉权重。
5. 其余模块必须有明显尺度差和位置差。
6. 必须有图标、符号或设备轮廓占位结构。
7. 必须有边框 / 编号 / 标签系统中的至少两种。
8. 可以使用 image 元素，但不能让整页沦为“图片拼贴”。
9. 必须保留标题区和说明区。
10. 页面必须像设备类型图谱，不像后台产品介绍。

推荐构图：
- 左侧标题和说明，右侧拼贴模块
- 上方大主模块，下方碎片化次级模块
- 一个垂直条带作为分类轴

禁止事项：
- 不要普通四卡片
- 不要功能介绍页
- 不要咨询模板
- 不要平均对齐
- 不要只有文本没有符号

只输出 JSON，不要 Markdown，不要解释。
```
