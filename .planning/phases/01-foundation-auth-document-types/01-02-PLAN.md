  <action>
    CRITICAL: Do NOT use @elysiajs/jwt. Do NOT use cookies for auth. Use Bearer Token + sessions table per CONTEXT.md decision.

    **auth.service.ts — Auth business logic with session management:**
    - `validateCredentials(username: string, password: string)`: Query users table by username, check isActive, verify password with Bun.password.verify. Return user object (id, username, displayName, role) on success, null on failure.
    - `createSession(userId: string)`: Generate token with `crypto.randomUUID()`. Insert into sessions table with token, userId, createdAt=now, expiresAt=now+7 days. Return the token string.
    - `getSessionUser(token: string)`: Query sessions table joined with users table where token matches AND expiresAt > now AND user.isActive is true. Return user object (id, username, displayName, role) or null. This is the core session lookup.
    - `deleteSession(token: string)`: Delete session record by token. Used for logout.
    - `deleteUserSessions(userId: string)`: Delete ALL session records for a user. Used when disabling a user — immediately revokes all access.
    - `getUserById(id: string)`: Query users table by id, return user object (id, username, displayName, role, isActive) or null.

    **auth.guard.ts — Auth middleware using ElysiaJS derive (reads Authorization header, NOT cookies):**
    - `authPlugin`: new Elysia({ name: 'auth' }). Use `derive` to:
      1. Read the Authorization header from the request: `request.headers.get('Authorization')`
      2. Extract token: if header starts with 'Bearer ', take the rest; otherwise null
      3. If token exists, call `getSessionUser(token)` to look up session in DB
      4. Return `{ user: sessionUser || null, sessionToken: token || null }` in context
    - `requireAuth`: new Elysia({ name: 'requireAuth' }), use authPlugin, guard with beforeHandle — if !user, return 401 { error: 'Unauthorized' }.
    - `requireAdmin`: new Elysia({ name: 'requireAdmin' }), use authPlugin, guard with beforeHandle — if !user return 401, if user.role !== 'admin' return 403 { error: 'Forbidden' }.
    - Export all three: authPlugin, requireAuth, requireAdmin.
    - IMPORTANT: No JWT imports, no cookie access, no @elysiajs/jwt usage anywhere.

    **auth.routes.ts — Auth endpoints (token in response body, NOT in cookies):**
    - POST /login: Accept { username, password } (validate with t.Object). Call validateCredentials. On failure return 401 { error: 'Invalid credentials' }. On success, call createSession(user.id) to get token. Return { token, user: { id, username, displayName, role } } in response body. No cookie setting.
    - GET /me: Use authPlugin derive. If no user in context, return 401 { error: 'Unauthorized' }. Return { user: { id, username, displayName, role } }.
    - POST /logout: Use authPlugin derive. If sessionToken exists, call deleteSession(sessionToken). Return { success: true }. No cookie clearing needed.
    - All routes under prefix '/auth'.

    **Update index.ts:**
    - Import authRoutes from modules/auth/auth.routes
    - Add .use(authRoutes) to the Elysia app
    - Ensure `export type App = typeof app` is AFTER all .use() calls so Eden Treaty captures all routes

    **api/client.ts — Eden Treaty client with Authorization header:**
    - Import { treaty } from '@elysiajs/eden'
    - Import type { App } from '@intelliflow/backend' (type-only import)
    - Configure Eden Treaty with a custom fetch that adds Authorization header:
      ```
      export const api = treaty<App>(window.location.origin, {
        fetch: {
          credentials: 'omit',  // No cookies
        },
        headers: () => {
          const token = localStorage.getItem('auth_token')
          return token ? { Authorization: `Bearer ${token}` } : {}
        }
      })
      ```
    - NOTE: Eden Treaty supports a `headers` option that can be a function returning headers object. This ensures every request includes the token automatically.
    - No cookies, no credentials: 'include' — pure Authorization header approach.
