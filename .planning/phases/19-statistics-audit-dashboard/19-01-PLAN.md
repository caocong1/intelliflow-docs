<task type="auto">
  <name>Task 2: Statistics service + routes</name>
  <files>packages/backend/src/modules/statistics/statistics.service.ts, packages/backend/src/modules/statistics/statistics.routes.ts, packages/backend/src/index.ts</files>
  <action>
1. Create `statistics.service.ts` with these functions:

   **Helper: `buildFilterConditions(filters)`** — builds an array of drizzle `and()` conditions from { dateFrom, dateTo, projectId, documentTypeId, workflowId }. Joins model_call_logs -> documents -> projects as needed. Department filtering uses `projects.department`.

   **`getOverviewKpis(filters)`** — Returns object with:
   - `totalCalls`: COUNT(*) from model_call_logs
   - `totalTokens`: SUM of `(token_usage->>'total_tokens')::int` with COALESCE to 0
   - `activeUsers`: COUNT(DISTINCT userId)
   - `docCount`: COUNT(DISTINCT documentId)
   - `estimatedCost`: SUM of cost calculated as: `COALESCE(budget_used_usd::numeric, 0) + COALESCE((token_usage->>'prompt_tokens')::numeric / 1000000 * models.input_price_per_mtok::numeric, 0) + COALESCE((token_usage->>'completion_tokens')::numeric / 1000000 * models.output_price_per_mtok::numeric, 0)` — LEFT JOIN with models for pricing
   - `todayCalls`: COUNT(*) WHERE created_at >= today's start
   - `avgDuration`: AVG(duration)
   - `avgSuccessRate`: COUNT(status=completed) / COUNT(*) * 100

   **`getTrends(filters, granularity)`** — GROUP BY `date_trunc(granularity, created_at)` returning period, callCount, totalTokens, estimatedCost per period. Order by period ASC.

   **`getByModel(filters)`** — GROUP BY modelId, modelName returning callCount, totalTokens, successRate, estimatedCost. Order by callCount DESC.

   **`getByUser(filters)`** — GROUP BY userId, JOIN users for name/displayName. Return callCount, docCount (COUNT DISTINCT documentId), totalTokens, estimatedCost. Order by callCount DESC.

   **`getByWorkflow(filters)`** — JOIN documents -> workflows. GROUP BY workflowId. Return workflowName, callCount, userCount (COUNT DISTINCT userId), docCount, estimatedCost. Order by callCount DESC.

   **`getModelTrends(filters, granularity)`** — Like getTrends but also grouped by modelId/modelName, for per-model trend charts.

   **`getUserTrends(filters, granularity)`** — Like getTrends but grouped by userId.

   **`getWorkflowTrends(filters, granularity)`** — Like getTrends but grouped by workflowId.

   **`getAuditByUser(filters, page, pageSize)`** — Query model_call_logs grouped by userId, joined with users table. Return list of users with aggregated stats (callCount, docCount, totalTokens, estimatedCost). Support pagination.

   **`getAuditByDocument(filters, page, pageSize)`** — Query model_call_logs joined with documents, users, workflows. Group by documentId. Return list of documents with: documentName, userName, workflowName, totalCalls, totalTokens, totalDuration, estimatedCost, createdAt. Support pagination. For expand detail: query model_call_logs WHERE documentId = X, returning each model call with: nodeName (from nodeExecutions), modelName, tokenUsage breakdown, duration, cost, responseStatus.

2. Create `statistics.routes.ts`:
   - Use `new Elysia({ prefix: "/admin/statistics" })`
   - Guard with admin role check (follow pattern from model-call-log.routes.ts)
   - Query params: `dateFrom?: string, dateTo?: string, granularity?: "day"|"week"|"month", projectId?: string, documentTypeId?: string, workflowId?: string, department?: string`
   - Endpoints:
     - `GET /overview` → getOverviewKpis
     - `GET /trends` → getTrends
     - `GET /by-model` → getByModel + getModelTrends (return both in one response)
     - `GET /by-user` → getByUser + getUserTrends
     - `GET /by-workflow` → getByWorkflow + getWorkflowTrends
     - `GET /audit/by-user` → getAuditByUser (paginated, accepts shared filters + page/pageSize)
     - `GET /audit/by-document` → getAuditByDocument (paginated, accepts shared filters + page/pageSize)
     - `GET /audit/document-detail/:documentId` → returns individual model calls for a document (for row expansion)

3. Register in `packages/backend/src/index.ts`: import and `.use(statisticsRoutes)`.
