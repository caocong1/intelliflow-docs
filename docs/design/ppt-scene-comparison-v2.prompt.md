# PPT Scene Single-Slide Prompt: Comparison V2

Model:

`gemini-3.1-pro-preview`

Goal:

生成一页高质量、高设计感的对比页，输出完整 `ppt_scene/v1` JSON，但 `slides` 数组中只能有 1 页。

Prompt:

```text
基于已提供的 `ppt_scene/v1` 协议，输出一个完全合法的 JSON。

任务：
- 只生成 1 页
- 页类型：comparison
- 主题：无线网络建设全流程指南
- 受众：企业与公共机构的采购负责人、IT负责人
- 语言：zh-CN

对比对象：
- 无线网络
- 传统有线网络

比较维度必须覆盖：
- 部署弹性
- 建设成本
- 扩容效率
- 适用场景

质量目标：
- 必须像高端品牌 deck / 展览展板 / 杂志对页
- 必须比当前常见 PPT 模板更大胆
- 第一眼必须像“在做抉择”，不是在读说明书

视觉方向：
- exhibition
- editorial comparison
- asymmetric
- sculptural layout
- high contrast
- premium

必须满足的设计要求：
1. 只输出一个完整的 `ppt_scene/v1` JSON。
2. `slides` 中必须且只能有 1 页。
3. 禁止 `table` 元素。
4. 禁止“左右两侧 + 中间4个 pill 标签 + 规则重复四行文本”这种模板化结构。
5. 必须有一个主视觉骨架，优先从以下中选择一种：
   - 对角切分
   - 中央展板
   - 偏移中轴
   - 左重右轻结构
6. 左右两侧不能机械镜像。
7. 必须包含至少一个大尺度背景结构 shape。
8. 必须包含至少一个“视觉锚点”，例如大字、水印、角标、徽章、编号。
9. 必须出现边框或框体语言，不允许全页只有纯色块。
10. 必须包含至少一组图标或图标占位结构。若没有真实图标资产，可使用几何 shape 组合成 icon badge。
11. 文本必须短句化，不能长段。
12. 页面必须保留可编辑性，不允许截图式处理。

推荐构图：
- 一侧为大标题 / 宣言，另一侧为信息阵营
- 中央为冲突结构，不要是规整标签列表
- 用 2-4 个不等尺寸信息块，而不是 4 行重复块

禁止事项：
- 不要 Word 表格
- 不要咨询模板双栏
- 不要普通左右对照
- 不要后台 UI
- 不要 4 组完全等高等宽信息条
- 不要输出多余说明

只输出 JSON，不要 Markdown，不要解释。
```
