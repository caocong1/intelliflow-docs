  <action>
1. Create `ValidationOverlay.tsx`:
   - Positioned below or above the canvas (as per user decision: "above/below canvas")
   - Collapsible panel showing validation errors after save
   - Each error item: severity icon (red circle for error, yellow triangle for warning) + message text + node label (if nodeId present)
   - Clicking an error item:
     a. Sets selectedNodeId to the error's nodeId (opens config panel)
     b. Calls fitView/setCenter to pan the canvas and center on the error node
   - Error count badge in the panel header: "校验错误 (3)"
   - "关闭" button to dismiss the overlay
   - Panel auto-shows when validation returns errors

2. Update all 5 custom node components:
   - Accept a `hasError` prop (boolean, derived from validation errors matching this node's ID)
   - When hasError is true: add `border-red-500 border-2` class to the node card (red highlighted border)
   - Animate the border (pulse or solid red) for visibility

3. Update WorkflowEditor.tsx save flow:
   - On "保存" button click:
     a. Call `PUT /api/workflows/:id` to save nodes/edges
     b. Call `POST /api/workflows/:id/validate` to get validation results
     c. Store validation errors in a signal
     d. If no errors: toast "保存成功" + set validationErrors to empty
     e. If errors: show ValidationOverlay with the errors, toast "保存成功，但流程存在校验问题"
   - Compute `errorNodeIds` from validation errors (set of nodeId values)
   - Pass errorNodeIds to WorkflowCanvas which passes hasError prop to each node

4. Add "草稿保存" behavior: workflow always saves regardless of validation (status stays "draft" or current). Validation errors are informational. Only enabling to "active" requires passing validation (handled by backend in 03-01).

5. Add data flow annotations on edges:
   - Update DataFlowEdge or WorkflowCanvas to show small labels on edges indicating what data flows (output names from source node)
   - If the source node has outputs defined, show a small tag/label on the edge midpoint with the output name(s)
   - Keep it subtle (small font, semi-transparent background) so it doesn't clutter the canvas
   - This satisfies FLOW-13 "visual preview displays node flow and file data paths"
