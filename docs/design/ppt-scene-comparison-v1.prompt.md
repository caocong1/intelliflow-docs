# PPT Scene Single-Slide Prompt: Comparison

Model:

`gemini-3.1-pro-preview`

Goal:

生成一页高质量对比页，输出完整 `ppt_scene/v1` JSON，但 `slides` 数组中只能有 1 页。

Prompt:

```text
基于已提供的 `ppt_scene/v1` 协议，输出一个完全合法的 JSON。

任务：
- 只生成 1 页
- 页类型：comparison
- 主题：无线网络建设全流程指南
- 受众：企业与公共机构的采购负责人、IT负责人
- 语言：zh-CN

内容目标：
对比“无线网络”与“传统有线网络”，至少覆盖以下维度：
- 部署弹性
- 建设成本
- 扩容效率
- 适用场景

质量目标：
- 必须像展板 / 杂志对页 / 品牌 deck 的对比页
- 不能像 Word 表格
- 不能像咨询模板中的双栏 bullet 对照
- 必须有明显设计张力

视觉方向：
- exhibition
- contrast
- axis-driven
- asymmetric
- editorial comparison

强制要求：
1. 只输出一个完整的 `ppt_scene/v1` JSON。
2. `slides` 中必须且只能有 1 页。
3. 禁止使用 `table` 元素。
4. 禁止输出左右完全镜像的四行条带。
5. 必须有一个清晰的“中轴 / 对角 / 分割结构”，作为主构图骨架。
6. 左右两侧阵营的视觉重量必须不同，不能机械对称。
7. 必须让比较维度成为中间组织结构，而不是小标题列表。
8. 必须以 shape + text 为主。
9. 文本要短句化，不能堆成长段。
10. 页面必须让人第一眼就看出“冲突 / 对抗 / 抉择”。

建议构图方向：
- 中央标签柱 + 左右阵营
- 对角切分 + 两侧论点
- 中轴药丸标签 + 两侧异形文本块

禁止事项：
- 不要普通表格
- 不要双栏说明书
- 不要标题下面四行一模一样的卡片
- 不要像后台 UI
- 不要输出多余说明

只输出 JSON，不要 Markdown，不要解释。
```
