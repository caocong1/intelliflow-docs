  <name>Task 1: Refactor Word export with state-machine parser for tables, lists, and code blocks</name>
  <files>packages/backend/src/modules/runtime/export.service.ts</files>
  <action>
Refactor `parseMarkdownToParagraphs()` into `parseMarkdownToElements()` that returns `(Paragraph | Table)[]` using a state machine with states: NORMAL, IN_TABLE, IN_CODE_BLOCK.

**State machine transitions:**
- NORMAL + line starting with `|` -> IN_TABLE (accumulate rows)
- IN_TABLE + line NOT starting with `|` -> NORMAL (flush table via `createWordTable()`, process line normally)
- NORMAL + line starting with ``` (triple backtick) -> IN_CODE_BLOCK (accumulate code lines)
- IN_CODE_BLOCK + line starting with ``` -> NORMAL (flush code block via `createCodeBlock()`)

**New helper functions:**

1. `createWordTable(headers: string[], rows: string[][]): Table` — Business-formal style per user decision:
   - Full borders: `BorderStyle.SINGLE`, size 1, color "999999"
   - Header row: bold text (via `parseInlineFormatting`), gray background `ShadingType.SOLID` color "E8E8E8", `tableHeader: true`
   - Data rows: alternating shading (odd rows: no fill, even rows: "F5F5F5")
   - Column widths: equal DXA distribution `Math.floor(9000 / colCount)`
   - Use `WidthType.DXA` for cell widths

2. `parseMarkdownTable(lines: string[]): { headers: string[]; rows: string[][] }` — Parse accumulated pipe-delimited lines:
   - Split each line by `|`, trim cells, filter empty edge cells
   - Identify separator row matching `/^\|[\s\-:|]+\|$/` and skip it
   - Lines before separator = headers, lines after = body rows

3. `createCodeBlock(lines: string[]): Paragraph[]` — Gray background monospace:
   - Each line as a `Paragraph` with `TextRun({ text: line, font: { name: "Courier New" }, size: 18 })`
   - Paragraph shading: `{ type: ShadingType.SOLID, color: "F3F4F6" }`

**Ordered list detection** in NORMAL state:
- Match `/^(\s*)(\d+)\.\s+(.+)/` for ordered list items
- Calculate nesting level from leading whitespace: 0-1 spaces = level 0, 2-4 = level 1, 5+ = level 2
- Use `numbering: { reference: "ordered-list", level: N }`

**Nested unordered list detection** in NORMAL state:
- Match `/^(\s+)[-*]\s+(.+)/` (indented bullet) for nested bullets
- 2-4 spaces = level 1, 5+ = level 2
- Existing top-level `- ` / `* ` detection stays at level 0

**Update `generateWordBuffer()`:**
- Add `numbering` config to Document constructor for ordered lists:
  ```
  numbering: { config: [{ reference: "ordered-list", levels: [
    { level: 0, format: "decimal", text: "%1.", alignment: AlignmentType.START, style: { paragraph: { indent: { left: 720, hanging: 360 } } } },
    { level: 1, format: "decimal", text: "%1.%2.", alignment: AlignmentType.START, style: { paragraph: { indent: { left: 1440, hanging: 360 } } } },
    { level: 2, format: "decimal", text: "%1.%2.%3.", alignment: AlignmentType.START, style: { paragraph: { indent: { left: 2160, hanging: 360 } } } },
  ] }] }
  ```
- Change `children: parseMarkdownToParagraphs(content)` to `children: parseMarkdownToElements(content)`
- Update docx imports to include `Table, TableRow, TableCell, BorderStyle, ShadingType, WidthType, AlignmentType`

**Preserve all existing behavior:** headings (h1/h2/h3), top-level bullets, blank lines, inline bold/italic formatting.

- Ordered lists: prefix with "N. " text, indent based on nesting level (level 0: 20px, level 1: 40px, level 2: 60px)
- Nested bullets: indent based on level, use bullet character at appropriate indent
- Strip markdown bold/italic from all PDF text (existing behavior, extend to new elements)
