### Integration Issues

**BROKEN-01: Workflow validation overlay response shape mismatch**
- Backend (`workflows.routes.ts`) returns `{ valid, errors }`
- Frontend (`WorkflowEditor.tsx`) expects nested `data.data.errors`
- Result: Validation errors never displayed in overlay
- Severity: Medium — validation still enforced on `PATCH /status` (enable), so invalid workflows can't be activated
- Affected: FLOW-10 (validation still works server-side, display broken client-side)

**MISSING-01: providerName absent from listActiveModels response**
- `listActiveModels()` returns only `modelColumns` (no JOIN to providers)
- `ModelCallConfig.tsx` reads `m.providerName` for optgroup label — always `undefined`
- Fallback: displays `m.providerId` (UUID) as group label
- Severity: Low — functional but poor UX

**MISSING-02: Shared Model type stale**
- `@intelliflow/shared` `Model` interface missing `temperature`, `maxTokens`, `topP`
- `ModelConfiguration.tsx` uses local `Model` type (lines 22-34) bypassing shared type
- Severity: Low — no runtime impact since shared type isn't consumed
