  <action>
    **users.service.ts — User CRUD logic:**
    - `listUsers(page: number, pageSize: number)`: Query users table, paginated, ordered by createdAt desc. Return { data: User[], total: number }. Exclude passwordHash from results.
    - `createUser(input: { username, password, displayName, role })`: Hash password with Bun.password.hash (argon2id). Insert into users table. Return created user (without passwordHash). Throw if username already exists.
    - `updateUser(id: string, input: { displayName?, role? })`: Update user fields, set updatedAt to now(). Return updated user. Throw if user not found.
    - `toggleUserStatus(id: string)`: Toggle isActive field. Return updated user. Throw if user not found. Prevent deactivating the last active admin.

    **users.routes.ts — Admin user management endpoints:**
    - All routes use requireAdmin guard, prefix '/users'
    - GET /users: Query params page (default 1), pageSize (default 20). Returns { data: User[], total, page, pageSize }.
    - POST /users: Body { username: string(3-50), password: string(6-100), displayName: string(1-100), role: enum('admin','user') }. Validate with ElysiaJS t.Object. Returns 201 with created user.
    - PATCH /users/:id: Body { displayName?: string, role?: enum }. Returns updated user.
    - PATCH /users/:id/status: Toggle isActive. Returns updated user.
    - Do NOT include DELETE — users are disabled, not deleted (per AUTH-02 "停用").
    - Handle unique constraint violations on username with 409 Conflict response.

    **document-types.service.ts — Document type CRUD logic:**
    - `listDocumentTypes(page: number, pageSize: number, search?: string)`: Query documentTypes table, paginated, ordered by createdAt desc. If search provided, filter by name OR code using ilike. Return { data, total }.
    - `createDocumentType(input: { name, code, description? })`: Insert into documentTypes. Return created record. Throw if code already exists.
    - `updateDocumentType(id: string, input: { name?, code?, description? })`: Update fields, set updatedAt. Return updated record.
    - `toggleDocumentTypeStatus(id: string)`: Toggle isActive. Return updated record.
    - `deleteDocumentType(id: string)`: Check for associated documents (for now, no documents table exists, so always allow — add the association check placeholder with a TODO comment for Phase 4 when documents table exists). Delete the record. Return { success: true }.

    **document-types.routes.ts — Admin document type endpoints:**
    - All routes use requireAdmin guard, prefix '/document-types'
    - GET /document-types: Query params page, pageSize, search (optional string). Returns { data, total, page, pageSize }.
    - POST /document-types: Body { name: string(1-100), code: string(1-50, pattern: alphanumeric + hyphen + underscore), description?: string(0-500) }. Returns 201.
    - PATCH /document-types/:id: Body { name?, code?, description? }. Returns updated.
    - PATCH /document-types/:id/status: Toggle isActive. Returns updated.
    - DELETE /document-types/:id: Returns 200 { success: true } or 409 if has associated documents.
    - Handle unique constraint on code with 409 Conflict.

    **Update index.ts:**
    - Import and .use(userRoutes) and .use(documentTypeRoutes)
    - Ensure export type App is after all .use() calls
