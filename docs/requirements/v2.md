# AI 文档生成平台 — 需求大纲（v2）

> 基于 2026-03-17 会议反馈，将原"多模型并行生成"架构升级为"流程编排"架构。

## 一、产品概述

### 1.1 产品定位

面向公司内部多部门使用的 AI 文档生成平台。用户以「项目」为单位组织工作，在项目内创建各类文档（招投标文档、解决方案、需求说明、技术方案等），通过**可编排的流程节点**驱动 AI 多模型并行生成、对比、迭代，快速产出高质量文档。

### 1.2 核心价值

- **流程可编排**：管理员用内置节点灵活编排文档生成流程，适配不同业务场景（招投标、技术方案、会议纪要等），无需开发介入
- **效率提升**：一次输入，多模型同时生成，省去逐个平台粘贴对比的重复劳动
- **质量保障**：内置质检、合规检查、一致性校验等节点，确保输出质量；多模型输出横向对比取各家之长
- **安全可控**：信息脱敏/恢复机制确保敏感信息不上传给 AI，映射关系本地存储
- **知识沉淀**：流程模板、生成历史全部留存，形成组织资产
- **协作共享**：项目成员共享文档与生成记录，避免信息孤岛

### 1.3 目标用户


| 角色    | 典型场景              |
| ----- | ----------------- |
| 售前/商务 | 招投标文档、解决方案、商务报价说明 |
| 产品经理  | PRD、竞品分析、用户调研报告   |
| 技术人员  | 技术方案、架构设计文档、接口说明  |
| 项目经理  | 项目计划、风险评估、会议纪要    |
| 管理层   | 战略规划、分析报告         |

## 二、功能模块

### 模块一：用户管理

#### 2.1.1 账号与认证（首选方案：企业微信）

系统作为企业微信内部 Web 应用接入，完全依赖企业微信完成用户认证和组织管理，不单独建设账号体系和组织架构管理页面：

- 用户从企业微信工作台点击应用入口，自动完成 OAuth 认证登录，无需手动输入账号密码
- 首次登录时自动创建系统账号，同步企业微信用户信息（姓名、头像、部门、职位等）
- 后续登录自动识别身份，静默完成认证
- 企业微信外部浏览器访问时，支持企业微信扫码登录
- 用户信息和部门数据以企业微信通讯录 API 为主数据源，不提供独立组织管理页面，但系统保留必要的用户/部门缓存快照用于审计和历史追溯
- 需要查询用户或部门时（如项目邀请成员），直接调用企业微信接口检索
- 离职人员的历史操作记录保留，展示"已离职"标记

> **备选方案**：如未来需要脱离企业微信独立部署，可扩展为独立账号体系（邮箱/手机号注册、账号密码登录、SSO 单点登录）和独立的组织部门管理模块。

#### 2.1.2 角色与权限

系统预设以下角色（可根据需要扩展）：


| 角色    | 权限范围                       |
| ----- | -------------------------- |
| 系统管理员 | 全局配置、文档类型管理、流程编排、模型配置、审计日志 |
| 普通用户  | 创建/参与项目、按流程生成文档、管理自己的内容    |

### 模块二：项目管理

#### 2.2.1 项目基本操作

- 创建项目（名称、描述、所属部门）
- 编辑项目信息
- 删除项目（软删除，管理员可恢复）
- 项目列表（我创建的 / 我参与的 / 部门项目 / 全部项目）
- 项目搜索与筛选（按名称、部门、创建时间）

#### 2.2.2 项目成员管理

- 邀请成员加入项目（通过企业微信通讯录搜索用户 / 按部门批量添加）
- 移除项目成员
- **成员自行退出项目**（成员可主动退出，退出后失去项目访问权限，该成员创建的文档归属不变）
- 项目内角色：


| 项目角色  | 权限                                                     |
| ----- | ------------------------------------------------------ |
| 项目负责人 | 项目全部权限：管理成员、项目配置、资料库管理、删除项目，以及**查看项目内所有文档**、创建/编辑/生成文档 |
| 项目参与者 | 创建文档、编辑/生成/查看**对自己可见的文档**（见 2.7.6 文档可见性），资料库的查看/上传/编辑  |


- 项目创建人默认为项目负责人
- 可添加多个项目负责人
- 查看项目成员列表及其角色

#### 2.2.3 项目主页

根据用户角色展示不同视图：

- **项目负责人视图**：顶部菜单包含「文档列表」「待办任务」「成员管理」「资料库」「项目设置」，默认展示文档列表
- **项目参与者视图**：顶部菜单包含「文档列表」「待办任务」「资料库」，默认展示文档列表
- **待办任务视图**：展示当前用户在本项目中待处理的任务（待审核的文档、被驳回待修改的文档等），按紧急程度和时间排序

#### 2.2.4 项目资料库

项目内置的资料管理中心，用于存放与本项目相关的参考资料（如招标文件、旧版方案、行业报告等）。流程中通过「选择资料」节点从资料库中选择参考资料。

- **资料上传**：支持上传 PDF / Word / Excel / 文本等常见格式
- **自动处理**：
  - 文件上传后自动进行内容解析（提取文本内容）
  - 自动进行 RAG 处理（文本切片、向量化索引），使 AI 生成时可精准检索相关段落
- **资料管理**：
  - 资料列表查看（文件名、大小、上传人、上传时间、处理状态）
  - 处理状态：上传中 / 解析中 / 就绪 / 解析失败
  - 资料预览（查看解析后的文本内容）
  - 删除资料（进入回收站，可恢复）
  - 资料支持版本更新（上传新版本，旧版本保留为历史版本）
- **回收站**：删除的资料进入回收站，项目负责人可在回收站中恢复或彻底删除
- **资料复用**：同一项目内的所有文档均可引用资料库中的资料，避免重复上传

### 模块三：文档类型管理

> 文档类型是文档的分类标签，定义"这是什么类型的文档"。不再包含模板、提示词、质量标准等配置（这些已下沉到流程模板的节点配置中）。

#### 2.3.1 文档类型管理

- 创建文档类型
- 编辑文档类型
- 启用 / 停用文档类型（停用后不可新建该类型文档，已有文档不受影响）
- 删除文档类型（仅无关联文档时可删除）
- 文档类型列表与搜索
- 文档类型排序（拖拽排序或权重排序）

#### 2.3.2 文档类型定义

每个文档类型包含以下配置项：


| 配置项  | 说明              | 示例                            |
| ---- | --------------- | ----------------------------- |
| 类型名称 | 文档类型的显示名称       | 「招投标技术方案」                     |
| 类型编码 | 唯一标识符           | `bid_technical_proposal`      |
| 类型描述 | 该类型文档的用途说明      | 「用于针对招标文件中的技术要求，生成结构化的技术响应方案」 |
| 默认流程 | 新建该类型文档时默认选用的流程 | 「招投标完整流程」                     |


#### 2.3.3 文档类型与流程的关系

- **所有流程对所有文档类型开放**，不做强绑定
- 每个文档类型可设置一个**默认流程**（新建文档时自动推荐）
- 用户新建文档时看到的流程选择顺序：
  1. 该类型的默认流程（推荐标记）
  2. 该类型历史使用过的流程（按使用频次排序）
  3. 所有其他可用流程

### 模块四：流程编排配置

> 全新核心模块。管理员在此配置文档生成的完整流程——从内置节点中选择、排序、连接，编排成适合特定业务场景的工作流。

#### 2.4.1 预设流程模板

每套模板包含该场景可能用到的**全部节点**，用户只需关闭不需要的节点即可。

> **脱敏分层原则**：信息脱敏在所有 AI 处理之前，信息恢复在所有 AI 处理之后。脱敏段内的所有节点（包括用户在这些节点上的编辑操作）均基于脱敏后内容；信息恢复之后的节点（人工审核、文件导出）及恢复后的编辑操作基于真实内容。

**模板1：通用文档生成（全功能版）** — 覆盖全部 12 个节点，不确定用哪个模板时选这个

```
信息录入
  → 信息脱敏
  ┌─── 所有AI处理（脱敏后内容）───────────┐
  │ → 选择资料                          │
  │ → 大纲确认                          │
  │ → AI生成                           │
  │ → AI质检                           │
  │ → 合规检查                          │
  │ → 一致性检查                        │
  │ → 查重检测                          │
  └───────────────────────────────────┘
  → 信息恢复
  → 人工审核
  → 文件导出
```

必选节点（不可关闭）：信息录入、AI生成、文件导出。典型裁剪示例：

- 不涉及敏感信息 → 关闭 信息脱敏 + 信息恢复
- 不需要参考资料 → 关闭 选择资料
- 短文档 → 关闭 大纲确认
- 不需要质检 → 关闭 AI质检/合规检查/一致性检查/查重检测
- 不需要审批 → 关闭 人工审核

> 用户在任意节点都可以**当场编辑**内容、**回退**到之前的节点重新执行、**跳过**后续AI步骤直接进入人工处理阶段。无需专门的"人工编辑"或"选择下一步"节点。

**模板2：招投标专用流程** — 增加 AI生成: 提取要求 步骤提取招标要求

```
信息录入(上传招标文件+填写项目信息)
  → 信息脱敏
  ┌─── 所有AI处理（脱敏后内容）───────────┐
  │ → AI生成: 提取技术要求           │
  │ → 选择资料(选历史中标方案)             │
  │ → 大纲确认                           │
  │ → AI生成: 生成投标响应                │
  │ → AI质检(对比参数一致性)              │
  │ → 合规检查(检查招标强制要求)          │
  │ → 一致性检查                         │
  │ → 查重检测                           │
  └──────────────────────────────────────┘
  → 信息恢复
  → 人工审核
  → 文件导出
```

**模板3：方案/报告生成流程** — 适用于技术方案、解决方案、分析报告等

```
信息录入 → 选择资料 → 大纲确认 → AI生成
  → 一致性检查 → 人工审核 → 文件导出
```

**模板4：快速生成流程** — 最简流程，适合邮件、通知等简单文档

```
信息录入 → AI生成 → 文件导出
```

**模板5：会议纪要/音视频处理流程** — 从录音、录像中生成文档

```
信息录入(上传音频/视频) → AI生成: 提炼要点 → 人工审核 → 文件导出
```

| 信息脱敏  | ✓     | ✓   |      |      |      |

| 大纲确认  | ✓     | ✓   | ✓    |      |      |
| AI生成  | ✓     | ✓×3 | ✓    | ✓    | ✓    |
| AI质检  | ✓     | ✓   |      |      |      |
| 合规检查  | ✓     | ✓   |      |      |      |
| 一致性检查 | ✓     | ✓   | ✓    |      |      |
| 查重检测  | ✓     | ✓   |      |      |      |
| 信息恢复  | ✓     | ✓   |      |      |      |
| 人工审核  | ✓     | ✓   | ✓    |      | ✓    |

- 系统管理员可在线新增预设模板
- 系统预设模板确保节点组合的合理性

#### 2.4.2 流程管理

- **基于模板创建流程**：选择一个预设流程模板，填写流程名称和描述，生成流程实例
- 编辑流程（在模板基础上裁剪）
- 启用 / 停用流程（停用后不可用于新建文档，已在使用的文档不受影响）
- 删除流程（仅无关联文档时可删除）
- 流程列表与搜索
- 流程复制（基于已有流程快速创建变体）

#### 2.4.3 流程模板编辑（系统管理员）

系统管理员可在线新增和编辑流程模板（节点配置直接内嵌在流程模板中，不单独管理）：

- 基于已有模板复制后修改，或从零创建新模板
- 编排节点：从 12 种内置节点中选择、排序、配置连接关系
- **直接配置每个节点的具体参数**：
  - 信息录入节点：配置输入表单字段（字段名、类型、是否必填、默认值）
  - AI 生成节点：配置提示词（System Prompt + User Prompt）、变量定义（变量名、类型、来源）、步骤说明文字、可用模型范围
  - 审核/质检节点：配置审核维度、检查项列表、AI 审核提示词
  - 导出节点：配置导出格式模板文件、内容映射规则
  - 大纲确认节点：配置大纲生成提示词、默认章节结构
- 变量系统：提示词中可使用 `{{变量名}}` 插值，变量可由上游节点输出自动填充或由用户手动填写
- 流程校验（系统自动检查）：
  - "信息恢复"前必须有"信息脱敏"（成对出现）
  - 流程必须有至少一个出口
- 流程预览：可视化展示节点流转图

#### 2.4.4 流程裁剪（普通用户）

普通用户新建文档选择流程模板后，可对流程进行**轻量裁剪**：

- **只能关闭节点**：对不需要的节点点击"跳过"使其变为灰色，执行时自动跳过该节点
- **不能增加节点**、不能调整顺序、不能修改节点间连接关系
- 部分节点不可关闭（由模板定义为"必选"，如"信息录入"和"文件导出"通常为必选）
- 管理员可在流程模板中将节点配置为**"必经"**（不可跳过，流程执行时用户无法跳过该节点）或**"可选"**（允许跳过）。所有跳过操作记录在审计日志中
- "信息脱敏"和"信息恢复"联动——关闭其中一个，另一个自动关闭
- 裁剪后的配置可保存为个人常用配置，下次新建同类文档时自动应用

#### 2.4.3 内置节点类型（12 种）

每个节点对应用户操作的一个**页面步骤**。用户在页面上执行操作、点击"确认/下一步"后流转到下一个节点。

##### 收集信息类

**① 信息录入**

用户在此页面填写需求、上传文件。

- 页面展示可配置的输入区域组合：文字输入框、结构化表单字段（公司名/项目编号/日期等）、文件上传区域
- 上传文件支持 Word / PDF / 图片 / 音频 / 视频，系统自动解析为文本（PDF/Word 提取文字，图片 OCR，音视频语音转文字）
- 用户可查看文件解析结果并修改
- 确认后：输入数据按模板定义的格式结构化为 JSON，流转给下游节点

**② 选择资料**

用户从项目资料库中挑选参考资料。

- 页面展示资料库浏览界面，支持搜索筛选和内容预览
- 用户勾选相关参考资料
- **跨项目引用**：
  - **项目负责人**可在此节点浏览自己作为负责人的其他项目的资料库，选择资料作为参考
  - 如果其他项目有文档被分享给当前用户（见 2.7.6 文档可见性），该文档也会出现在可选列表中，可作为参考资料引用
  - 跨项目引用为只读引用，不复制文件到当前项目
- 确认后：系统对选中资料进行 RAG 检索（向量检索相关段落），检索结果作为上下文传递给下游 AI 节点

##### AI 处理类

**③ AI 生成**

核心节点——调用 AI 生成内容。每个 AI 生成节点在流程中的具体用途由其关联的**节点配置**决定（如"提取招标要求""起草投标响应""提炼会议要点"等），用户无需关心底层实现——只需看到步骤说明、补充必要信息、点击生成。

- 页面分为三个区域：
  - **步骤说明区**：展示本步骤的任务描述（来自节点配置，如"根据招标要求生成投标响应"）+ 来自上游节点的输入数据摘要，用户可展开查看完整上下文
  - **变量填写区**：节点配置中定义的变量，部分由上游数据自动填充，未填充的变量以表单形式展示让用户补充（如无需填写则此区域不显示）
  - **生成区**：模型选择面板 + 生成结果展示区
- 提示词对用户**默认隐藏**（已由节点配置配置好），高级用户可点击「查看/编辑提示词」展开
- **模型选择**：默认「系统推荐」1 个最优模型，用户可展开切换为「多模型对比」模式手动多选。模型展示增加标签（速度优先 / 质量优先 / 低成本），并显示预估生成耗时
- 各模型通过 SSE 流式输出，每个模型有独立输出卡片，状态依次为：等待中 → 思考中(动画) → 生成中(内容流式输出) → 已完成/失败
- 支持切换查看不同模型输出（Markdown 渲染视图 / 源码视图）
- 单个模型失败可单独重试，已成功模型的输出保留不受影响
- 多模型时支持左右分栏对比
- **引用来源展示**：生成结果中标注引用来源（来自哪份参考资料的哪个段落），用户可查看每段内容的引用依据；导出时可选附带参考来源附录；用户可剔除某份资料后重新生成
- 用户选定最佳输出作为本节点最终输出
- 确认后：选定的输出流转给下游节点，所有模型输出记录保留

> **同一流程中可包含多个 AI 生成节点**，各自关联不同的节点配置，承担不同任务。例如招投标流程中依次有三个 AI 生成节点：第一个的模板是"从招标文件提取技术要求"，第二个是"根据要求起草投标响应"，第三个是"交叉参考评分标准优化内容"。用户在每个步骤只需看到任务说明、补充信息、点击生成。节点的具体配置由系统管理员在流程模板中直接设定（见 2.4.3）。

##### 确认类

**④ 大纲确认**

AI 先列大纲/目录结构，用户确认后再生成正文。

- 页面展示 AI 根据输入信息自动生成的文档大纲（章节标题列表）
- 用户可拖拽调整章节顺序、增删章节、修改标题
- 确认后：大纲结构流转给下游 AI 生成节点，AI 按此结构逐章生成

##### 审核把关类

**⑤ 人工审核**

指定人员审核内容，通过或打回。

- 流程到达此节点时，系统自动**通知审核人**（通过企业微信消息/系统内通知），审核任务出现在审核人的「我的待办」列表中（见 2.7.1）
- 审核人从待办列表点击进入审核页面，页面展示待审核文档内容 + 审核表单（管理员在节点配置中预设审核维度，如"技术方案完整性""数据准确性"等）
- 审核人逐项检查，标记通过/不通过，填写审核意见
- 可在文档内容上直接批注或小幅修改
- 确认后：
  - 通过 → 继续流转到下游节点，通知文档创建人
  - 驳回 → 驳回时**必须填写审核意见**；流程回退到管理员配置的目标节点（如 AI 生成），携带审核意见，通知文档创建人
  - 有条件通过 → 审核意见随文档继续流转
- 审核人配置：节点配置中指定审核人规则（项目负责人 / 指定角色 / 文档创建人的直属上级 / 手动指定）

**⑥ AI 质检**

让 AI 按标准自动检查，出检查报告。

- AI 按节点配置定义的审核标准自动检查，生成审核报告（各维度得分/评级、问题清单、修改建议）
- 特别能力：对比招标参数与文档内容的一致性（检查招标文件要求的每项指标在投标文件中是否有响应）
- 用户阅读报告，决定是否通过
- 确认后：通过 → 继续流转；不通过 → 携带报告回退到指定节点

##### 质量检查类

**⑦ 查重检测**

检测内容重复率。

- 页面展示文档内容（重复段落高亮标记）+ 查重报告（总重复率、各段落重复来源）
- 用户查看高亮标记的重复段落和来源
- 如重复率过高，可直接编辑修改或回退重新生成
- 确认后：达标 → 继续流转；不达标 → 回退修改

**⑧ 合规检查**

对照标准/招标要求，逐条检查有没有漏项或不合规。

- 页面展示合规校验报告，每条规则的检查结果（通过/不通过/需关注）
- 校验规则由节点配置定义（如"所有技术指标必须有量化数据""必须包含风险评估章节"等）
- 对不通过的项目查看详细说明和修改建议
- 确认后：继续流转或回退修改

**⑨ 一致性检查**

检查全文前后有没有矛盾。

- 页面展示一致性检查报告，列出文档中自相矛盾或不一致的地方
- 如："第二章提到项目周期为6个月，但第四章里程碑计划跨度为8个月""前文用'客户'，后文用'甲方'，术语不统一"
- 每个不一致项展示具体位置和上下文
- 确认后：继续流转或回退修改

##### 安全处理类

**⑩ 信息脱敏**

把敏感信息替换为占位符，防止上传给线上云端 AI。

- 使用**本地私有部署的 AI 模型**识别文档中的敏感信息（公司名/人名/金额/联系方式/地址等）并高亮标记
- 用户逐条确认是否需要脱敏，选择脱敏方式（替换为占位符如"XX公司" / 删除 / 泛化如具体金额→"XX万元"）
- 可手动标记 AI 未识别的敏感信息
- **原始信息↔占位符的映射关系本地存储**，不随内容上传给线上 AI
- 多轮回退后映射关系保持一致（映射表与脱敏节点执行版本绑定）
- 确认后：脱敏后的内容流转给下游节点，后续所有 AI 节点处理的都是脱敏内容

**⑪ 信息恢复**

所有 AI 处理完成后，把占位符替换回真实信息。

- **纯本地处理，不调用任何 AI 模型**——读取本地存储的脱敏映射，自动将文档中的占位符替换回原始敏感信息
- 页面展示恢复前后的对比（高亮标记已恢复的位置）
- 用户确认恢复结果是否正确
- 用户手动编辑占位符导致恢复失败时：高亮标记失败项，允许用户手动修正
- 确认后：恢复后的完整文档流转给下游节点（人工审核、文件导出），确保人看到的是真实内容

##### 输出交付类

**⑫ 文件导出**

选择格式，预览并导出下载。

- 选择导出格式（Word .docx / PDF / Excel .xlsx / Markdown .md）
- 选择导出排版模板（如公司标准 Word 模板）
- 预览导出效果
- 设置文件名
- 可选择导出哪些内容（如果流程中有多个内容产出）
- 点击「导出」下载文件
- 确认后：文件生成并下载，流程可结束或继续到后续节点

##### 每个节点的通用操作能力

所有节点除了各自的专属功能外，均支持以下四种通用操作：


| 操作         | 说明                                                                                      |
| ---------- | --------------------------------------------------------------------------------------- |
| **确认，下一步** | 正常流转到流程中的下一个节点                                                                          |
| **编辑当前内容** | 展开内联编辑器（Markdown编辑+预览），当场修改当前节点的输出后再继续。支持 AI 辅助编辑（信息恢复前可用线上模型，恢复后仅限本地模型）                |
| **跳过后续**   | 跳过后续链路中的**可选节点**，自动流转至最近的下一个必经节点（如信息恢复、人工审核）；若后续无必经节点则直接进入文件导出。跳过的可选节点标记为"已跳过"，不可跨越必经节点 |
| **回退**     | 回到之前的某个节点重新执行，后续节点状态重置为"待执行"。用户不满意 AI 生成结果时可直接回退重新生成，实现迭代                               |


> 通过这四种通用操作，用户可以在任意节点灵活地编辑内容、跳过不需要的步骤、回退重做——无需专门的"人工编辑"或"选择下一步"节点。

#### 2.4.4 节点间数据流

- 上游节点的输出自动作为下游节点的可用输入上下文
- 多个上游节点的输出可自动合并（管理员可配置合并顺序和分隔方式）
- 节点配置中的变量占位符由上游节点输出的数据自动填充
- 节点间的数据格式自动转换（如 JSON ↔ 文本）

#### 2.4.5 流程引擎后台能力

以下能力由流程引擎自动处理，不作为用户交互节点：


| 能力   | 说明                            |
| ---- | ----------------------------- |
| 条件路由 | 管理员配置的自动条件分支（如"字数>5000走长文流程"） |
| 版本快照 | 每个节点完成后自动保存版本记录               |
| 通知推送 | 特定节点完成时自动发送通知（可配置在任意节点上）      |
| 并行执行 | 管理员配置的并行分支（如多个章节同时生成）         |

### 模块五：AI 模型配置

#### 2.5.1 模型提供商（Provider）管理

支持多种模型提供商接入（如阿里云 DashScope、OpenAI 兼容接口、自定义 HTTP 接口等），具体 API 协议详见附录 B。

Provider 实例管理：

- 创建 Provider 实例（选择类型、填写名称、API 地址、认证信息）
- 编辑 Provider 实例
- 测试连通性（发送测试请求验证配置是否正确）
- 启用 / 停用
- 删除

#### 2.5.2 模型管理

每个 Provider 实例下可配置多个模型：

- 添加模型（模型 ID、显示名称、描述、部署类型、参数配置）
- 编辑模型信息
- 启用 / 停用模型
- 删除模型
- **部署类型**（必填）：


| 部署类型 | 说明                           | 可处理敏感信息 |
| ---- | ---------------------------- | ------- |
| 线上云端 | 调用外部云端 API（如阿里云 DashScope 等） | ✗       |
| 本地私有 | 部署在公司内网的模型服务                 | ✓       |


- 信息脱敏节点：仅可使用**本地私有**模型（因需处理敏感原文）
- 信息恢复节点：纯本地处理（读取映射替换），不调用 AI 模型
- 各节点"当场编辑"功能的 AI 辅助：在信息恢复之后，仅可使用**本地私有**模型
- 其他 AI 节点：在脱敏流程段内，线上/本地均可（内容已脱敏）
- 模型参数配置：
  - temperature（创造性）
  - max_tokens（最大输出长度）
  - top_p
  - 其他模型特定参数（以 JSON 形式扩展）

#### 2.5.3 用量与限制

- 按用户 / 部门 / 项目 / 模型设置每日 / 每月调用次数限制
- 按模型设置单次最大 Token 限制
- 用量统计面板（调用次数、Token 消耗，按模型 / 用户 / 部门 / 项目维度）
- 超限提醒与处理策略（提醒 / 阻断 / 允许超限）

### 模块六：文档工作台

> 用户日常操作的核心界面，取代旧版的"文档生成"模块。用户按流程逐节点执行，每个节点是一个操作页面。

#### 2.6.1 我的待办

系统全局入口，汇聚当前用户**跨项目**的所有待处理任务：

- **待审核**：其他成员提交到「人工审核」节点、等待当前用户审核的文档
- **被驳回**：当前用户创建的文档被审核人驳回、需要修改后重新提交的文档
- **进行中**：当前用户正在执行流程但尚未完成的文档

每条待办显示：文档标题、所属项目、文档类型、当前节点、提交人、提交时间。按紧急程度和时间排序，支持筛选。

点击待办直接进入对应文档的当前节点操作页面。

> 同样的待办信息在项目主页的「待办任务」标签页中按项目维度展示（见 2.2.3）。

#### 2.6.2 创建文档

在项目内新建文档，支持以下三种入口：

- **从空白创建**（默认方式）：
  1. 选择文档类型（从已启用的类型列表中选择）
  2. 选择流程（系统自动推荐最匹配的流程并标记"推荐"，每个流程附带一句话适用场景说明；展示顺序：该类型的默认流程(推荐) → 历史使用过的流程 → 所有其他可用流程）
  3. 可选：点击「自定义流程」展开流程裁剪选项（默认折叠）
  4. 填写文档基本信息（文档标题、描述/备注）
  5. 进入文档工作台
- **从已有文档导入**：上传 Word / PDF 文件，系统解析内容后进入流程，用户可选择从哪个节点开始执行（如跳过信息录入，直接进入大纲确认或 AI 优化）
- **从历史文档复制**：基于项目内已有文档创建副本，在副本上继续加工

#### 2.6.3 工作台界面

- **流程进度导航栏**（顶部）：展示当前流程的所有节点，标记状态（已完成 ✓ / 进行中 → / 待执行 ○）
- **当前节点操作区**（主体）：根据节点类型展示对应的操作界面（参见 2.4.3 各节点描述）
- **节点历史面板**（侧边）：查看已完成节点的输入输出记录
- 可回退到已完成的节点查看历史或重新执行（重新执行后，后续节点状态重置为"待执行"）

#### 2.6.4 流程执行

- 用户在每个节点页面上完成操作后，点击「确认/下一步」流转到下一个节点
- 用户在任意节点可点击「回退」回到之前的节点重新执行，实现迭代
- 到达「文件导出」节点时，用户可下载文件，流程可结束
- 到达「人工审核」节点时，流程暂停等待审核人操作
- 支持多模型流式输出（技术协议详见附录 A）

#### 2.6.5 失败恢复与任务连续性

- **自动保存**：每个节点的操作过程自动保存草稿（表单输入、编辑内容），无需用户手动保存
- **断点续作**：浏览器关闭或网络中断后，重新进入文档可恢复到上次操作状态
- **后台执行**：AI 生成支持后台执行，用户可离开页面，生成完成后通过企业微信消息和系统内通知提醒用户
- **部分失败处理**：多模型生成时，已成功的模型输出保留，失败的可单独重试
- **取消生成**：支持取消正在进行的 AI 生成任务

#### 2.6.6 通知规则

- 默认通知场景：
  - 待审核：文档流转到人工审核节点时，通知审核人
  - 被驳回：文档被审核人驳回时，通知文档创建人
  - 后台生成完成：AI 后台生成完成时，通知文档创建人
- 通知渠道：企业微信消息 + 系统内待办列表

#### 2.6.7 全局工作台增强（后续迭代）

> 以下能力在后续版本中逐步支持：

- 全局搜索：跨项目搜索文档、资料、流程
- 最近访问：最近打开的文档列表
- 收藏：收藏项目、文档、常用流程
- 常用流程快捷创建：一键使用常用流程新建文档

### 模块七：文档管理

#### 2.7.1 文档列表

- 项目内文档列表（按创建时间 / 类型 / 状态 / 使用流程 筛选和排序）
- 列表展示信息：文档标题、文档类型、使用流程、创建人、创建时间、当前节点、状态
- 文档搜索（按标题、描述关键词搜索）
- 文档状态：草稿 / 流程执行中 / 待审核 / 已完成 / 待复审
- 批量操作（批量删除，删除后进入回收站）

#### 2.7.2 文档详情

- 查看文档基本信息（标题、类型、使用流程、创建人、创建时间、最后更新时间）
- **流程执行历史**：展示每个节点的执行记录，每条记录可查看：
  - 该节点使用的模板
  - 用户的输入内容
  - 节点的输出结果（如 AI 生成的内容、审核意见等）
  - 使用的 AI 模型（如有）
  - 执行耗时
- 如流程有迭代（回退重新执行），展示迭代历史（迭代 1 / 迭代 2 / ...），可切换查看

#### 2.7.3 内容编辑

- 从流程中任意 AI 生成节点的输出中，选定一个版本标记为「最终版本」
- 基于最终版本可进行两种编辑方式：
  - **手动编辑**：在 Markdown 编辑器中直接修改内容，支持实时预览
  - **AI 辅助修改**：输入修改提示词，选择某个模型，AI 基于当前内容进行修改，流式输出修改结果
- AI 辅助修改的结果可选择「采纳」（替换最终版本）或「放弃」
- 每次编辑保存都生成新的版本记录，可回溯

#### 2.7.4 版本管理

- 流程中每个节点完成时自动生成版本快照
- 用户在任意节点"当场编辑"保存也生成新版本
- 版本列表查看（时间线形式）
- 版本对比（选择两个版本进行 Diff 查看）
- 版本回退（将某个历史版本恢复为当前版本）

#### 2.7.5 导出

AI 生成的内容统一为 Markdown 格式，导出时转换为目标格式：

- 导出为 Word 文档（`.docx`）
- 导出为 Excel 表格（`.xlsx`）
- 导出为 PDF 文件（`.pdf`）
- 导出为 Markdown 文件（`.md`，原始格式）
- 可选择导出 AI 原始输出或用户编辑后的最终版本
- 批量导出（选择多个文档打包下载）

#### 2.7.6 文档可见性

文档创建人可设置文档的可见范围：


| 可见范围     | 说明                          |
| -------- | --------------------------- |
| 仅自己可见    | 只有文档创建人可查看和编辑（**新建文档默认选项**） |
| 项目成员可见   | 项目内所有成员可查看                  |
| 指定成员可见   | 仅创建人指定的项目成员可查看              |
| 指定项目外成员可见 | 可将文档分享给项目外的系统用户（通过企业微信通讯录搜索指定），被分享者可查看该文档并在自己的项目中将其作为参考资料引用（见 ② 选择资料节点） |


- **项目负责人**始终可查看项目内所有文档，不受可见性设置限制
- 文档可见性定义**查看范围**；文档的编辑、继续生成、提交审核等操作仅限**文档创建人**和**项目负责人**执行，其他可见成员默认为只读
- 审核中的文档内容**锁定**，创建人不可编辑（需审核人驳回后才能修改）
- 已完成的文档如需再编辑，状态自动变为「待复审」，需重新走审核流程

#### 2.7.7 文档评论与批注（后续版本）

> 后续版本支持文档评论与行内批注能力，实现更轻量的协作反馈。

### 模块八：统计与审计

#### 2.8.1 总览面板

- 总调用次数、总 Token 消耗、活跃用户数、文档生成数
- 总对账金额

#### 2.8.2 生成记录明细

每次文档生成记录完整的审计信息：


| 记录维度     | 内容            |
| -------- | ------------- |

| 文档信息     | 文档标题、文档类型     |

#### 2.8.3 模板使用统计

## 三、核心使用流程

### 3.1 管理员初始化流程

```
管理员登录系统
    │
    ├── 1. 配置模型提供商与模型
    │   ├── 添加 Provider 实例（如 DashScope）
    │   ├── 填写 API 地址和认证信息
    │   ├── 测试连通性
    │   └── 在 Provider 下添加具体模型
    │
    ├── 2. 创建文档类型
    │   └── 创建文档类型（名称、编码、描述、默认流程）
    │
    ├── 3. 创建流程
    │   ├── 选择预设流程模板（如「招投标完整流程」「标准生成」等）
    │   ├── 填写流程名称和描述
    │   ├── 在模板基础上裁剪（关闭不需要的节点）
    │   ├── 配置每个节点的具体参数（提示词、表单字段、审核维度等）
    │   ├── 为 AI 生成节点配置可用模型范围
    │   ├── 系统自动校验流程合理性
    │   └── 启用流程
    │
    └── 4. 用户管理（企业微信自动同步）
```

### 3.2 用户日常使用流程

```
用户登录系统
    │
    ├── 1. 进入或创建项目
    │   ├── 从「我的项目」列表选择已有项目
    │   └── 或创建新项目，邀请相关成员
    │
    ├── 2. 上传项目资料（可在任意时间进行）
    │   ├── 进入项目资料库
    │   ├── 上传相关参考文件
    │   └── 系统自动解析、RAG 处理，等待状态变为「就绪」
    │
    ├── 3. 新建文档
    │   ├── 选择文档类型
    │   ├── 选择流程模板（默认推荐 / 历史使用 / 全部可用）
    │   ├── 可选：关闭不需要的节点（轻量裁剪）
    │   ├── 填写文档标题
    │   └── 进入文档工作台
    │
    ├── 4. 按流程逐节点执行
    │   ├── 每个节点是一个操作页面
    │   ├── 完成操作后点击「确认/下一步」流转
    │   ├── 可在任意节点当场编辑内容（支持AI辅助）
    │   ├── 不满意时可回退到之前的节点重新执行，实现迭代
    │   └── 满意时可跳过后续步骤直接进入导出
    │
    └── 6. 导出
        ├── 在导出节点选择格式并下载
        └── 或在文档管理中随时导出
```

### 3.3 文档生成的完整数据流（以招投标为例）

```
信息录入（上传招标文件 + 填写项目信息）
    ──→ 信息脱敏（本地AI识别，替换敏感信息为占位符，映射本地存储）
    ┌─── 以下所有AI处理均使用脱敏后内容 ────────────────────────┐
    │ ──→ AI 生成: 提取技术要求                                  │
    │ ──→ 选择资料（从资料库选择历史中标方案）                      │
    │ ──→ 大纲确认（生成投标响应大纲，用户调整确认）                │
    │ ──→ AI 生成: 生成投标响应（按大纲+要求+参考资料，多模型生成）  │
    │     ※ 用户可在此处回退重新生成，实现迭代                     │
    │     （多模型输出在AI生成节点内直接对比选优）                    │
    │ ──→ AI 质检（逐条对比招标参数与投标文档的一致性）              │
    │ ──→ 合规检查（检查是否满足招标强制要求）                      │
    │ ──→ 一致性检查（检查全文数据/术语一致性）                     │
    │ ──→ 查重检测（检测重复率）                                   │
    │     ※ 用户可在任意节点当场编辑内容、跳过后续直接进入恢复       │
    └──────────────────────────────────────────────────────────┘
    ──→ 信息恢复（本地处理，占位符→真实信息）
    ──→ 人工审核（部门负责人审核，看到真实完整内容）
    ──→ 文件导出（导出 Word 文档）
```

### 3.4 项目协作流程

```
项目负责人创建项目
    │
    ├── 邀请团队成员（参与者）、可添加其他负责人
    │
    ├── 参与者 A：新建「技术方案」文档 → 选择流程 → 按流程执行 → 导出
    ├── 参与者 B：新建「商务报价」文档 → 选择流程 → 按流程执行 → 导出
    │
    ├── 成员可自行退出项目
    │
    └── 项目负责人：
        ├── 同样可创建/编辑/生成文档
        ├── 管理成员、项目配置、资料库
        └── 查看项目统计数据
```

## 四、非功能需求概述

> 仅列出关键的非功能需求方向，不涉及具体技术实现。


| 维度  | 要求                                                     |
| --- | ------------------------------------------------------ |
| 性能  | 流式生成响应延迟 < 2 秒（首 Token）；支持 50+ 用户同时生成                  |
| 安全  | 数据传输加密；API Key 等敏感信息加密存储；接口鉴权；脱敏映射本地存储不上传              |
| 可用性 | 单模型失败不影响其他模型；单节点失败不影响流程中其他节点；服务异常可快速恢复                 |
| 数据  | 生成记录完整保留；流程执行历史可审计；支持数据备份与恢复                           |
| 易用性 | 界面简洁直观，核心操作路径不超过 3 步；流程编排提供可视化拖拽界面                     |
| 扩展性 | 新 Provider 类型可通过扩展代码添加；文档类型、流程、模板均可在线配置；新节点类型可通过扩展代码添加 |

## 五、关键业务规则

> 集中说明产品中的关键边界规则，避免研发和业务理解不一致。


| #   | 规则             | 说明                                                   |
| --- | -------------- | ---------------------------------------------------- |
| 1   | 文档不支持多人同时编辑    | 同一文档同一时间只允许一个用户编辑，其他人只能查看                            |
| 2   | 审核中文档锁定        | 文档进入人工审核节点后，内容锁定不可编辑，需审核人驳回后创建人才能修改                  |
| 3   | 已完成文档再编辑触发复审   | 已完成的文档如需再编辑，状态自动变为「待复审」，需重新走审核流程                     |
| 4   | 删除统一走回收站       | 文档和资料的删除均为软删除，进入回收站，项目负责人可恢复                         |
| 5   | 项目删除为软删除       | 项目删除后其文档和资料保留，系统管理员可恢复                               |
| 6   | 支持文档另存为        | 用户可将文档「另存为新文档」，在副本上继续修改，不影响原文档                       |
| 7   | 流程模板变更不影响进行中文档 | 管理员修改流程模板后，已在执行中的文档继续使用创建时的流程快照，不受影响                 |
| 8   | 跨项目不可复制文档，但可引用 | 文档与项目绑定，不支持跨项目复制；但项目负责人可跨项目引用资料，被分享的文档可被项目外成员作为参考资料只读引用（见 ② 选择资料） |
| 9   | 资料引用版本固定       | 文档在流程执行中引用资料时，固定引用到当时的资料版本快照；资料后续更新不影响历史文档的引用结果与审计追溯 |

AI 生成节点中多模型生成的流式通信协议：


| 事件类型          | 说明       | 数据              |

### 附录 B：Provider 类型与 API 风格


| Provider 类型   | 说明                     | API 风格                  |
| ------------- | ---------------------- | ----------------------- |
| 阿里云 DashScope | 阿里旗下模型网关               | OpenAI 兼容接口             |

每种 Provider 类型为系统内置代码实现，封装了：

- API 请求格式构建
- 流式响应解析
- 错误处理与重试策略
- 认证方式（API Key / Token 等）
