**export.service.ts:**

1. `generateExport(documentId: string, nodeExecutionId: string, format: "word" | "pdf" | "markdown", content: string, filename: string, userId: string)`:
   - Resolve content from upstream node outputs (similar to prompt template resolution — use contentMapping from config to pull from prior node outputData)
   - Generate file based on format:
     - **Markdown**: write content directly as `.md` file (simplest)
     - **Word (.docx)**: use `docx` npm package (`bun add docx`). Create a Document with Paragraph elements from markdown-parsed content. Convert markdown headings to HeadingLevel, bold/italic to TextRun with formatting, lists to bullet paragraphs. Use Packer.toBuffer() to generate.
     - **PDF**: use `pdfkit` npm package (`bun add pdfkit`). Create PDFDocument, add content as text with basic formatting (headings = larger font, body = 12pt). Pipe to buffer.
   - Save generated file to `getExportPath(documentId)/${filename}`
   - Call `insertDocumentFile` to index in DB with category="export"
   - Update nodeExecution.outputData = { format, filename, storagePath, fileSize }
   - Return { filename, storagePath, fileSize, format }

2. `getExportPreview(documentId: string, nodeExecutionId: string)`:
   - Return the source content (markdown text) that will be exported
   - For preview purposes, the frontend renders this as markdown

3. `downloadExport(documentId: string, nodeExecutionId: string)`:
   - Read the generated file from disk
   - Return file buffer with appropriate Content-Type and Content-Disposition headers

**export.routes.ts:**
Routes under `/runtime/:documentId/export/:nodeExecutionId`:
- `GET /preview` — returns { content, defaultFilename } (markdown source content + suggested filename like "{title}_{date}.{ext}")
- `POST /generate` — body: { format, filename }, calls generateExport, returns file metadata
- `GET /download` — returns binary file with Content-Disposition: attachment

Register in runtimeRoutes.
