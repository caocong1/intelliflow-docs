# AI 文档生成平台 — 需求大纲

## 一、产品概述

### 1.1 产品定位

面向公司内部多部门使用的 AI 文档生成平台。用户以「项目」为单位组织工作，在项目内创建各类文档（招投标文档、解决方案、需求说明、技术方案等），通过 AI 多模型并行生成、对比、迭代，快速产出高质量文档。

### 1.2 核心价值

- **效率提升**：一次输入，多模型同时生成，省去逐个平台粘贴对比的重复劳动
- **质量保障**：多模型输出横向对比，取各家之长；迭代优化持续打磨
- **知识沉淀**：文档类型、模板、提示词、生成历史全部留存，形成组织资产
- **协作共享**：项目成员共享文档与生成记录，避免pri信息孤岛

### 1.3 目标用户


| 角色    | 典型场景              |
| ----- | ----------------- |
| 售前/商务 | 招投标文档、解决方案、商务报价说明 |
| 产品经理  | PRD、竞品分析、用户调研报告   |
| 技术人员  | 技术方案、架构设计文档、接口说明  |
| 项目经理  | 项目计划、风险评估、会议纪要    |
| 管理层   | 战略规划、分析报告         |

## 二、功能模块

### 模块一：用户管理

#### 2.1.1 账号与认证（首选方案：企业微信）

系统作为企业微信内部 Web 应用接入，完全依赖企业微信完成用户认证和组织管理，不单独建设账号体系和组织架构管理页面：

- 用户从企业微信工作台点击应用入口，自动完成 OAuth 认证登录，无需手动输入账号密码
- 首次登录时自动创建系统账号，同步企业微信用户信息（姓名、头像、部门、职位等）
- 后续登录自动识别身份，静默完成认证
- 企业微信外部浏览器访问时，支持企业微信扫码登录
- 用户信息和部门数据直接通过企业微信通讯录 API 实时获取，不做本地管理页面
- 需要查询用户或部门时（如项目邀请成员），直接调用企业微信接口检索

> **备选方案**：如未来需要脱离企业微信独立部署，可扩展为独立账号体系（邮箱/手机号注册、账号密码登录、SSO 单点登录）和独立的组织部门管理模块。

#### 2.1.2 角色与权限

系统预设以下角色（可根据需要扩展）：


| 角色    | 权限范围                  |
| ----- | --------------------- |
| 系统管理员 | 全局配置、文档类型管理、模型配置、审计日志 |
| 普通用户  | 创建/参与项目、生成文档、管理自己的内容  |

### 模块二：项目管理

#### 2.2.1 项目基本操作

- 创建项目（名称、描述、所属部门）
- 编辑项目信息
- 删除项目（软删除，管理员可恢复）
- 项目列表（我创建的 / 我参与的 / 部门项目 / 全部项目）
- 项目搜索与筛选（按名称、部门、创建时间）

#### 2.2.2 项目成员管理

- 邀请成员加入项目（通过企业微信通讯录搜索用户 / 按部门批量添加）
- 移除项目成员
- 项目内角色：


| 项目角色  | 权限                                             |
| ----- | ---------------------------------------------- |
| 项目负责人 | 项目全部权限：管理成员、项目配置、资料库管理、删除项目，以及所有文档的创建/编辑/生成/查看 |
| 项目参与者 | 所有文档的创建/编辑/生成/查看，资料库的查看/上传/编辑                  |


- 项目创建人默认为项目负责人
- 可添加多个项目负责人
- 查看项目成员列表及其角色

#### 2.2.3 项目主页

根据用户角色展示不同视图：

- **项目负责人视图**：顶部菜单包含「文档列表」「成员管理」「资料库」「项目设置」，默认展示文档列表
- **项目参与者视图**：顶部菜单包含「文档列表」「资料库」，默认展示文档列表

#### 2.2.4 项目资料库

项目内置的资料管理中心，用于存放与本项目相关的参考资料（如招标文件、旧版方案、行业报告等）。生成文档时从资料库中选择参考资料，而非每次单独上传。

- **资料上传**：支持上传 PDF / Word / Excel / 文本等常见格式
- **自动处理**：
  - 文件上传后自动进行内容解析（提取文本内容）
  - 自动进行 RAG 处理（文本切片、向量化索引），使 AI 生成时可精准检索相关段落
  - 自动进行脱敏处理（去除公司名称、客户名称、金额、联系方式等敏感信息），脱敏结果供上传者确认
- **资料管理**：
  - 资料列表查看（文件名、大小、上传人、上传时间、处理状态）
  - 处理状态：上传中 / 解析中 / 就绪 / 解析失败
  - 资料预览（查看解析后的文本内容）
  - 删除资料
- **资料复用**：同一项目内的所有文档均可引用资料库中的资料，避免重复上传

### 模块三：文档类型配置

> 文档类型不是系统内置的，而是可由管理员灵活配置的通用类型体系。

#### 2.3.1 文档类型管理

- 创建文档类型
- 编辑文档类型
- 启用 / 停用文档类型（停用后不可新建该类型文档，已有文档不受影响）
- 删除文档类型（仅无关联文档时可删除）
- 文档类型列表与搜索
- 文档类型排序（拖拽排序或权重排序）

#### 2.3.2 文档类型定义

每个文档类型包含以下配置项：


| 配置项   | 说明                | 示例                            |
| ----- | ----------------- | ----------------------------- |
| 类型名称  | 文档类型的显示名称         | 「招投标技术方案」                     |
| 类型编码  | 唯一标识符             | `bid_technical_proposal`      |
| 类型描述  | 该类型文档的用途说明        | 「用于针对招标文件中的技术要求，生成结构化的技术响应方案」 |
| 文档规范  | 模板、质量标准、参考文档等     | 见下方详细说明                       |
| 边界与限制 | 该类型文档不应包含的内容、注意事项 | 见下方详细说明                       |


#### 2.3.3 文档规范

> 将原「内容定义」和「模板管理」合并为统一的「文档规范」，避免两者在描述文档结构时的重叠。

每个文档类型通过以下三部分定义其文档规范：

**一、文档模板**

模板是文档结构的具体表达，以 Markdown 骨架形式呈现，包含章节标题和每个章节的内容说明。模板即内容定义——通过模板本身来体现该类型文档应包含哪些章节、每个章节应写什么。

- 一个文档类型可配置多个模板（适用于不同场景，如「标准版」「简版」）
- 创建模板（模板名称、适用说明、Markdown 骨架内容）
- 编辑、删除模板
- 设置默认模板（新建该类型文档时自动选用）
- 模板示例：
  ```
  # {项目名称} 技术方案

  ## 一、项目概述
  <!-- 说明项目背景、目标和范围 -->

  ## 二、技术架构
  <!-- 描述整体技术架构和关键技术选型理由 -->

  ## 三、实施计划
  <!-- 给出详细的实施步骤、里程碑和时间安排 -->

  ## 四、风险评估
  <!-- 识别主要风险并给出应对措施 -->
  ```

**二、质量标准**

定义该类型文档的质量要求，AI 生成时作为约束条件：

- 如「数据需有来源引用」「方案需包含风险评估」「每个功能模块需有用户场景描述」
- 支持添加多条质量标准规则

**三、参考文档**

上传该类型文档的优秀范例，作为 AI 生成时的内容参考：

- 支持上传多份参考文档（PDF / Word / 文本等格式）
- 上传后系统自动进行**脱敏处理**（去除公司名称、客户名称、金额、联系方式等敏感信息），脱敏后的内容供管理员确认
- 参考文档管理（查看、替换、删除）

#### 2.3.4 边界与限制

为每个文档类型定义「内容不应包含什么」以及其他约束：

- **禁止内容**：如「不得包含具体代码实现」「不得涉及商业敏感数据」
- **语言要求**：如「必须使用中文」「专业术语需附英文原文」
- **篇幅限制**：如「总字数建议 3000-5000 字」
- **格式约束**：如「表格不超过 10 行」「图片用文字描述代替」

#### 2.3.5 系统提示词

- 系统根据文档类型的配置信息（类型描述、模板、质量标准、参考文档、边界与限制）**自动生成**系统提示词
- 管理员可预览自动生成的提示词内容
- 如需微调，可切换到手动编辑模式进行修改
- 修改后可随时点击「重新生成」恢复为自动生成版本

#### 2.3.6 提示词优化配置

每个文档类型可单独配置提示词优化策略：

- 优化提示词（Optimization Prompt）：用于将用户原始输入改写为高质量提示词
- 优化使用的模型（可指定用某个特定模型做优化）
- 是否默认开启「自动优化」

### 模块四：AI 模型配置

#### 2.4.1 模型提供商（Provider）管理

系统内置多种 Provider 类型，每种类型对应一种 API 调用方式：


| Provider 类型   | 说明                     | API 风格                  |
| ------------- | ---------------------- | ----------------------- |
| 阿里云 DashScope | 阿里旗下模型网关               | OpenAI 兼容接口             |
| OpenAI 兼容     | 兼容 OpenAI API 格式的第三方服务 | OpenAI Chat Completions |
| 自定义 HTTP      | 自定义 HTTP 接口            | 自定义请求/响应映射              |


Provider 类型为系统内置代码实现，每种类型封装了：

- API 请求格式构建
- 流式响应解析
- 错误处理与重试策略
- 认证方式（API Key / Token 等）

Provider 实例管理：

- 创建 Provider 实例（选择类型、填写名称、API 地址、认证信息）
- 编辑 Provider 实例
- 测试连通性（发送测试请求验证配置是否正确）
- 启用 / 停用
- 删除

#### 2.4.2 模型管理

每个 Provider 实例下可配置多个模型：

- 添加模型（模型 ID、显示名称、描述、参数配置）
- 编辑模型信息
- 启用 / 停用模型
- 删除模型
- 模型参数配置：
  - temperature（创造性）
  - max_tokens（最大输出长度）
  - top_p
  - 其他模型特定参数（以 JSON 形式扩展）

#### 2.4.3 用量与限制

- 按用户 / 部门 / 项目 / 模型设置每日 / 每月调用次数限制
- 按模型设置单次最大 Token 限制
- 用量统计面板（调用次数、Token 消耗，按模型 / 用户 / 部门 / 项目维度）
- 超限提醒与处理策略（提醒 / 阻断 / 允许超限）

### 模块五：文档生成

> 核心模块，参考 Prism 的多模型并行生成架构。

#### 2.5.1 创建文档

在项目内新建文档：

1. 选择文档类型（从已启用的类型列表中选择）
2. 填写文档基本信息：
  - 文档标题
  - 文档描述 / 备注
3. 选择模板（该类型下如有多个模板可选择，否则使用默认模板）
4. 进入文档工作区

#### 2.5.2 文档工作区 — 输入区

文档工作区分为「输入区」和「输出区」：

**输入区功能：**

- **需求输入**：多行文本输入框，用户填写原始需求描述
- **参考资料选择**：从项目资料库中选择与本次生成相关的参考资料（如招标文件、旧版方案等），系统通过 RAG 检索资料中的相关内容作为 AI 生成时的上下文
- **提示词优化**：
  - 点击「优化提示词」按钮
  - 系统使用该文档类型配置的优化提示词和优化模型，将用户输入改写为高质量提示词
  - 流式展示优化过程（思考中 → 生成中）
  - 用户可选择「采用优化结果」或「忽略」
  - 优化前后的提示词均自动保存
- **模型选择**：
  - 展示系统中所有已启用的模型
  - 默认全选，用户可取消勾选（至少保留 1 个）
  - 显示每个模型的名称和简要描述
- **提示词历史**：
  - 展示当前文档的所有历史提示词（按时间倒序）
  - 每条记录标注来源：手动保存 / 优化前 / 优化后 / 生成时自动保存
  - 可点击任意历史提示词回填到输入框
  - 可删除历史提示词

#### 2.5.3 文档工作区 — 生成与输出区

**生成流程：**

1. 用户点击「生成文档」
2. 系统自动保存当前提示词（来源标记：生成时自动保存）
3. 创建一个新的「生成迭代」记录
4. 界面切换到输出区
5. 通过 SSE（Server-Sent Events）流式接收各模型的生成结果

**多模型串行生成：**

按选中的模型顺序依次生成（参考 Prism 的串行策略）：

```
模型 A：等待中 → 思考中（动画）→ 生成中（内容流式输出）→ 已完成
模型 B：等待中 → 思考中 → 生成中 → 已完成
模型 C：等待中 → 思考中 → 生成中 → 已完成
...
全部完成
```

**SSE 事件协议：**


| 事件类型          | 说明       | 数据              |
| ------------- | -------- | --------------- |
| `model_start` | 某模型开始生成  | 模型 ID           |
| `token`       | 流式内容片段   | 模型 ID + 内容文本    |
| `model_done`  | 某模型生成完成  | 模型 ID + 输出记录 ID |
| `model_error` | 某模型生成失败  | 模型 ID + 错误信息    |
| `all_done`    | 全部模型生成完成 | 无               |


**状态展示：**

每个模型在侧边栏显示实时状态：

- 等待中（灰色）
- 思考中（动画效果，表示模型正在推理）
- 生成中（内容正在流式输出）
- 已完成（绿色勾号）
- 失败（红色，附重试按钮）

**输出查看：**

- 单模型查看：点击侧边栏中的模型名称，右侧展示该模型的完整输出
- 切换视图：Markdown 渲染视图 / 原始 Markdown 源码视图
- 复制：一键复制当前模型输出的 Markdown 内容
- 对比查看：选择两个模型进行左右分栏对比

#### 2.5.4 单模型重试

- 当某个模型生成失败时，可针对该模型单独重试
- 仅重新调用失败模型的 API，其他模型结果不受影响
- 重试过程同样通过 SSE 流式返回

#### 2.5.5 对比报告与迭代

迭代的唯一路径：生成 → 对比报告 → 输入修改意见 → 下一轮生成。

- 在所有模型输出完成后，可选择 2 个及以上模型生成「对比分析报告」
- 报告内容：
  - 各模型输出的结构完整性对比
  - 内容深度与细节丰富度对比
  - 表达风格与专业性对比
  - 各模型优劣势总结
  - 综合推荐意见
- 报告同样通过 AI 生成，流式输出
- 用户在对比报告的基础上输入修改意见，发起下一轮迭代生成
- 迭代导航：顶部显示迭代列表（迭代 1 / 迭代 2 / 迭代 3 ...），可切换查看任意历史迭代
- 每次迭代独立记录：用户输入、修改意见、对比报告、各模型输出

#### 2.5.7 提示词组装逻辑

系统在三种场景下自动组装不同的提示词：

**场景一：首次生成文档**


| 组成部分          | 来源                          | 角色          |
| ------------- | --------------------------- | ----------- |
| System Prompt | 文档类型的系统提示词（自动生成或手动微调）       | system 消息   |
| 模板内容          | 用户选择的模板（如有）                 | user 消息的一部分 |
| 参考资料          | 从项目资料库中选择的资料（经 RAG 检索的相关段落） | user 消息的一部分 |
| 用户输入          | 用户填写的需求描述（可能经过优化）           | user 消息的一部分 |


**场景二：生成对比分析报告**


| 组成部分          | 来源                          | 角色          |
| ------------- | --------------------------- | ----------- |
| System Prompt | 对比分析专用系统提示词（系统内置）           | system 消息   |
| 各模型输出         | 用户选择的 2 个及以上模型的完整输出内容       | user 消息的一部分 |
| 分析要求          | 系统预设的对比维度（结构完整性、内容深度、表达风格等） | user 消息的一部分 |


**场景三：基于报告的迭代生成**


| 组成部分          | 来源                          | 角色          |
| ------------- | --------------------------- | ----------- |
| System Prompt | 文档类型的系统提示词                  | system 消息   |
| 模板内容          | 用户选择的模板（如有）                 | user 消息的一部分 |
| 参考资料          | 从项目资料库中选择的资料（经 RAG 检索的相关段落） | user 消息的一部分 |
| 原始用户输入        | 首次生成时的需求描述                  | user 消息的一部分 |
| 上轮模型输出        | 上一轮各模型的完整输出                 | user 消息的一部分 |
| 对比分析报告        | 对比报告的完整内容                   | user 消息的一部分 |
| 修改意见          | 用户补充的修改意见（如有）               | user 消息的一部分 |

### 模块六：文档管理

#### 2.6.1 文档列表

- 项目内文档列表（按创建时间 / 类型 / 状态筛选和排序）
- 列表展示信息：文档标题、文档类型、创建人、创建时间、迭代次数、状态
- 文档搜索（按标题、描述关键词搜索）
- 文档状态：草稿 / 生成中 / 已完成
- 批量操作（批量删除）

#### 2.6.2 文档详情

- 查看文档基本信息（标题、类型、创建人、创建时间、最后更新时间）
- 迭代历史列表：展示所有迭代记录，每条记录可查看：
  - 该次迭代使用的用户输入 / 提示词
  - 各模型的生成输出
  - 对比分析报告（如有）
  - 用户的修改意见（如有）
- 可切换查看任意迭代的完整内容

#### 2.6.3 内容编辑

- 从任意迭代的任意模型输出中，选定一个版本标记为「最终版本」
- 基于最终版本可进行两种编辑方式：
  - **手动编辑**：在 Markdown 编辑器中直接修改内容，支持实时预览
  - **AI 辅助修改**：输入修改提示词，选择某个模型，由 AI 基于当前最终版本内容进行修改，流式输出修改结果
- AI 辅助修改的结果可选择「采纳」（替换最终版本）或「放弃」
- 每次编辑保存都生成新的版本记录，可回溯

#### 2.6.4 版本管理

- 每次 AI 生成迭代自动生成版本
- 人工编辑保存也生成新版本
- 版本列表查看（时间线形式）
- 版本对比（选择两个版本进行 Diff 查看）
- 版本回退（将某个历史版本恢复为当前版本）

#### 2.6.5 导出

AI 生成的内容统一为 Markdown 格式，导出时转换为目标格式：

- 导出为 Word 文档（`.docx`）
- 导出为 Excel 表格（`.xlsx`）
- 导出为 PDF 文件（`.pdf`）
- 导出为 Markdown 文件（`.md`，原始格式）
- 可选择导出 AI 原始输出或人工编辑后的最终版本
- 批量导出（选择多个文档打包下载）

## 三、核心使用流程

### 3.1 管理员初始化流程

```
管理员登录系统
    │
    ├── 1. 配置模型提供商
    │   ├── 添加 Provider 实例（如 DashScope）
    │   ├── 填写 API 地址和认证信息
    │   ├── 测试连通性
    │   └── 在 Provider 下添加具体模型
    │
    ├── 2. 配置文档类型
    │   ├── 创建文档类型（如「招投标技术方案」）
    │   ├── 填写类型描述
    │   ├── 配置文档规范（创建模板、设置质量标准、上传参考文档）
    │   ├── 设置边界与限制（禁止内容、语言要求、篇幅限制）
    │   ├── 编写或自动组装系统提示词
    │   └── 配置提示词优化策略
    │
    └── 3. 用户与部门管理
        ├── 创建部门结构
        ├── 添加用户账号
        └── 分配角色
```

### 3.2 用户日常使用流程

```
用户登录系统
    │
    ├── 1. 进入或创建项目
    │   ├── 从「我的项目」列表选择已有项目
    │   └── 或创建新项目，邀请相关成员
    │
    ├── 2. 新建文档
    │   ├── 点击「新建文档」
    │   ├── 选择文档类型（如「解决方案」）
    │   ├── 填写文档标题
    │   └── 选择模板（可选）
    │
    ├── 2.5 上传项目资料（可在任意时间进行）
    │   ├── 进入项目资料库
    │   ├── 上传相关参考文件（如招标文件、旧版方案等）
    │   └── 系统自动解析、RAG 处理、脱敏，等待状态变为「就绪」
    │
    ├── 3. 填写需求 & 生成
    │   ├── 在输入区填写原始需求描述
    │   ├── 从项目资料库选择参考资料（可选）
    │   ├── 可选：点击「优化提示词」提升输入质量
    │   ├── 选择/取消选择模型（默认使用所有已启用模型）
    │   └── 点击「生成文档」
    │
    ├── 4. 查看生成结果
    │   ├── 实时观看各模型的流式生成过程
    │   ├── 切换查看不同模型的输出
    │   ├── 使用对比模式横向比较
    │   └── 如有模型失败，可单独重试
    │
    ├── 5. 迭代优化（可选）
    │   ├── 生成对比分析报告
    │   ├── 根据报告分析修改提示词
    │   ├── 重新生成（创建新迭代）
    │   └── 重复直到满意
    │
    ├── 6. 人工编辑
    │   ├── 选择最佳模型输出作为基础
    │   ├── 在编辑器中进行人工修改
    │   └── 保存为最终版本
    │
    └── 7. 导出与分享
        ├── 导出为 Word / PDF / Markdown
        └── 或生成分享链接发送给相关方
```

### 3.3 文档迭代的完整数据流

```
第 1 次生成：
    用户输入 ──→ [可选: 提示词优化] ──→ 系统提示词 + 模板 + 用户输入 + 资料库参考内容(RAG)
        ──→ 模型 A/B/C 生成 ──→ 输出 A₁/B₁/C₁

    ↓ 用户不满意

迭代（可重复多轮）：
    选择输出 A₁ 和 C₁ ──→ 生成对比报告 ──→ 用户输入修改意见
        ──→ 增强提示词（原始输入 + 模型输出 + 对比报告 + 修改意见）
        ──→ 模型 A/B/C 生成 ──→ 输出 A₂/B₂/C₂
        ──→ 继续对比 → 修改意见 → 下一轮 ... 直到满意

    ↓ 用户满意

选定最终版本：
    选定输出 A₃ 标记为最终版本 ──→ 手动编辑 / AI 辅助修改 ──→ 导出
```

### 3.4 项目协作流程

```
项目负责人创建项目
    │
    ├── 邀请团队成员（参与者）、可添加其他负责人
    │
    ├── 参与者 A：创建「技术方案」文档 → 生成 → 编辑 → 保存
    ├── 参与者 B：创建「商务报价」文档 → 生成 → 编辑 → 保存
    │
    └── 项目负责人：
        ├── 同样可创建/编辑/生成文档
        ├── 管理成员、项目配置、资料库
        └── 导出/分享最终文档
```

## 四、非功能需求概述

> 仅列出关键的非功能需求方向，不涉及具体技术实现。


| 维度  | 要求                                    |
| --- | ------------------------------------- |
| 性能  | 流式生成响应延迟 < 2 秒（首 Token）；支持 50+ 用户同时生成 |
| 安全  | 数据传输加密；API Key 等敏感信息加密存储；接口鉴权         |
| 可用性 | 单模型失败不影响其他模型；服务异常可快速恢复                |
| 数据  | 生成记录完整保留；支持数据备份与恢复                    |
| 易用性 | 界面简洁直观，核心操作路径不超过 3 步                  |
| 扩展性 | 新 Provider 类型可通过扩展代码添加；文档类型和模型可在线配置   |
