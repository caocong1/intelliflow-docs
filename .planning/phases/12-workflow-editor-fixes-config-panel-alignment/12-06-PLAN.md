**Step A: DataFlowEdge.tsx — animation, selection, hit area**

1. Add flow animation to the edge path:
   - Apply CSS class `edge-animated` with keyframe animation:
   ```css
   stroke-dasharray: 8 4;
   animation: edge-flow 0.6s linear infinite;
   ```
   - Define the keyframe via inline style or a `<style>` tag within the SVG:
   ```
   @keyframes edge-flow { from { stroke-dashoffset: 24; } to { stroke-dashoffset: 0; } }
   ```
   - Respect `prefers-reduced-motion`: disable animation if user prefers reduced motion

2. Add wider invisible hit area for edge selection:
   - Render an invisible path with `stroke-width="12"` and `stroke="transparent"` BEFORE the visible path
   - Both paths share the same `d` attribute
   - The invisible path handles click events

3. Selection highlight:
   - Accept `selected` prop (boolean) from edge data
   - When selected: increase visible stroke width from 2 to 3, change color to brighter indigo (#818CF8), add subtle glow via filter or thicker stroke

4. Keep existing data flow label annotation functionality

**Step B: WorkflowCanvas.tsx — MiniMap colors**

Update MiniMap to use per-type colors:
```typescript

**Step C: WorkflowCanvas.tsx — Alignment guides**

Implement basic alignment guides during node drag:
1. If SolidFlow exposes `onNodeDrag` or `onNodeDragStop`: use it to compute alignment positions
2. During drag, compare the dragging node's x/y center with all other nodes' x/y centers
3. If within a snap threshold (5px), render a temporary guide line (horizontal or vertical SVG line) at that position
4. Use a signal `alignmentGuides` that stores active guide positions: `Array<{ type: "horizontal" | "vertical"; position: number }>`
5. Render guides as absolutely positioned divs or SVG lines within the canvas container
6. Clear guides on drag end
7. If SolidFlow doesn't expose onNodeDrag, skip alignment guides (note in SUMMARY) — this is an enhancement, not a blocker

**Step D: Pass edge selection state**

  });

```
This requires WorkflowCanvas to receive `selectedEdgeId` as a prop or manage it internally (Plan 03 already adds it internally).
