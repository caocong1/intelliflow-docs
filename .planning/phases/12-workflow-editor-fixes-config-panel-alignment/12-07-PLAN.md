**1. Create `prompt-optimize.ts`** — New endpoint for prompt optimization:

Route: `POST /api/prompts/optimize`

Request body:
```typescript
{
  promptText: string;     // The current prompt to optimize
  modelId: string;        // Which model to use for optimization
  metaPrompt?: string;    // Optional custom optimization instruction
}
```

Response:
```typescript
{
  optimizedText: string;  // The optimized prompt
  modelUsed: string;      // Model ID that was used
}
```

Implementation:
- Validate request: `promptText` must be non-empty, `modelId` must exist in the models table
- Build the optimization prompt:
  - If `metaPrompt` is provided, use it as the system instruction
  - If not, use a default system meta-prompt (Claude's discretion on content, but something like):
    ```
    你是一个提示词优化专家。请优化以下提示词，使其更加清晰、具体、结构化。
    保留原始意图和所有变量引用（如 {{节点名.输出名}}），不要改变变量格式。
    只返回优化后的提示词文本，不要添加解释。
    ```
  - The user's `promptText` goes as the user message
- Call the model via the existing model call service (reuse the OpenAI-compatible API call pattern from the model call module)
- Return the optimized text
- Handle errors: model not found (404), model call failure (502), empty prompt (400)

**2. Create `index.ts`** (or update existing prompts module index):
- Export the router
- Register route in the main app router (check existing pattern — likely in `packages/backend/src/index.ts` or a routes aggregator)

Ensure the endpoint is protected by the existing auth middleware (Bearer token pattern used throughout the backend).
