**In packages/shared/src/types.ts — extend FormFieldDef interface:**

1. Expand `type` union to: `"text" | "textarea" | "file" | "number" | "date" | "datetime" | "select" | "multiselect"`
2. Add optional fields:
   - `machineKey?: string` — stable identifier, format `/^[a-zA-Z_][a-zA-Z0-9_]*$/`
   - `options?: string[]` — plain text option list for select/multiselect
   - `defaultValue?: string` — single default for text/number/date/datetime/select
   - `defaultValues?: string[]` — multiple defaults for multiselect only
3. Keep existing `fileCountMode` and `acceptedFileTypes` as-is (file-specific)

**In packages/backend/src/modules/workflows/validation.ts — add validation rules after the existing formFields validation block (after L184):**

Add a new validation block for input_transform fields that checks:

1. **machineKey format:** For each field with a `machineKey`, validate against `/^[a-zA-Z_][a-zA-Z0-9_]*$/`. Error message: `字段 "${field.label}" 的 machineKey "${field.machineKey}" 格式不合法，只允许英文字母、数字和下划线，且不能以数字开头`
2. **machineKey uniqueness:** Collect all machineKeys within the node's fields into a Set. If duplicate found, error: `字段 "${field.label}" 的 machineKey "${field.machineKey}" 与其他字段重复`
3. **select/multiselect options required:** If `field.type === "select" || field.type === "multiselect"`, require `field.options` to be a non-empty array. Error: `字段 "${field.label}" 需要至少一个选项`
4. **Option text no commas:** For select/multiselect fields, validate that no option text contains a comma (since multiselect stores comma-joined). Error: `字段 "${field.label}" 的选项不能包含逗号`
5. **Default value in options:** If `field.defaultValue` is set for select, check it exists in `field.options`. If `field.defaultValues` is set for multiselect, check each exists in `field.options`. Error: `字段 "${field.label}" 的默认值不在选项列表中`
  </action>
