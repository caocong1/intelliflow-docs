# PPT Scene Single-Slide Prompt: Summary V2

Model:

`gemini-3.1-pro-preview`

Goal:

生成一页高质量、高收束感的总结页，输出完整 `ppt_scene/v1` JSON，但 `slides` 数组中只能有 1 页。

Prompt:

```text
基于已提供的 `ppt_scene/v1` 协议，输出一个完全合法的 JSON。

任务：
- 只生成 1 页
- 页类型：summary
- 主题：无线网络建设全流程指南
- 受众：企业与公共机构的采购负责人、IT负责人
- 语言：zh-CN

总结的四个原则：
- 合规优先
- 场景适配
- 容量预估
- 可运维性

质量目标：
- 必须像最终结论海报，而不是内容页
- 必须有强收束感、强节奏、强层级
- 必须比当前“丝带条带”更成熟，避免重复感

视觉方向：
- poster
- premium
- asymmetric
- strong closure
- architectural rhythm
- brand deck ending

必须满足的设计要求：
1. 只输出一个完整的 `ppt_scene/v1` JSON。
2. `slides` 中必须且只能有 1 页。
3. 禁止四卡片。
4. 禁止楼梯式重复卡片。
5. 禁止 4 条几乎一样的 ribbon 机械重复。
6. 必须存在一个“大结论主体区”。
7. 四个原则必须通过以下至少两种方式形成节奏差异：
   - 长短差异
   - 粗细差异
   - 位置差异
   - 色彩权重差异
   - 编号层级差异
8. 必须包含边框、徽章、编号、角标、细分隔线中的至少两种装饰语言。
9. 必须包含一个“最后一句收束文案”区域。
10. 文本必须短、狠、结论化。
11. 不允许像中间页继续展开内容。

推荐构图：
- 左侧结论碑文 + 右侧不等长原则条带
- 中央结论牌 + 周边原则标签
- 主海报牌面 + 侧向编号系统

禁止事项：
- 不要 bullet list
- 不要普通总结页
- 不要平均分配四原则
- 不要四个完全一样的容器
- 不要输出多余说明

只输出 JSON，不要 Markdown，不要解释。
```
