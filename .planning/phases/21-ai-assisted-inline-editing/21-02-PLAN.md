**useTextSelection.ts:**

Create a SolidJS hook `useTextSelection(textareaRef: () => HTMLTextAreaElement | undefined)` that returns a reactive signal `SelectionState | null`.

```typescript
interface SelectionState {
  text: string;
  start: number;
  end: number;
  // Position relative to the textarea's container for toolbar placement
  rect: { top: number; left: number; width: number };
}
```

Implementation:
- Listen to `mouseup` and `keyup` events on the textarea (not `selectionchange` on document -- that fires too frequently and doesn't reliably fire for textareas in all browsers).
- On each event, read `textarea.selectionStart` and `textarea.selectionEnd`.
- If start === end (no selection), set signal to null.
- Otherwise, compute rect from `textarea.getBoundingClientRect()` relative to offsetParent.
- **Critical pitfall prevention:** Store selection in signal immediately on mouseup/keyup. Toolbar buttons must use `onMouseDown` with `preventDefault()` to avoid textarea blur which would clear the selection.
- Clean up event listeners in `onCleanup`.

**sse-stream.ts:**

Extract the SSE streaming pattern from ModelCallExecutor into a reusable utility:

```typescript
interface SSEEvent {
  type: "delta" | "complete" | "error" | "status";
  modelId: string;
  data: string;
}

interface SSEStreamOptions {
  url: string;
  method?: "GET" | "POST";
  body?: unknown;
  onDelta: (modelId: string, data: string) => void;
  onComplete: (modelId: string, data: string) => void;
  onError: (modelId: string, data: string) => void;
  signal: AbortSignal;
}

export async function streamSSE(options: SSEStreamOptions): Promise<void>
```

Implementation: Follow the exact pattern from ModelCallExecutor.tsx lines 159-215:
- Get auth token from localStorage
- Use fetch with Authorization header and abort signal
- POST method: include `Content-Type: application/json` and `JSON.stringify(body)`
- Read response body with ReadableStream reader
- Split buffer on "\n\n", parse "data:" lines as JSON SSEEvent objects
- Call appropriate callback based on event.type
- Handle AbortError silently
