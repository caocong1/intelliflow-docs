  <name>Task 1: Extend VariablePicker with JSON Schema tree expansion</name>
  <files>packages/frontend/src/components/workflow/prompt/VariablePicker.tsx</files>
  <action>
Extend the existing VariablePicker to show expandable tree structure for upstream nodes that have `jsonSchema` configured.

**1. JSON Schema tree builder helper:**

Add a `buildSchemaTree(schema: object, parentPath: string)` function that recursively builds a tree structure from a JSON Schema object:

```typescript
interface SchemaTreeNode {
  key: string;           // property name
  path: string;          // full dot-path from root (e.g. "items[0].name")
  type: string;          // "object", "array", "string", "number", etc.
  children?: SchemaTreeNode[];
  isLeaf: boolean;
  isArray: boolean;
}
```

- For `type: "object"` with `properties`: create child nodes for each property
- For `type: "array"` with `items`: create `[0]` (fixed index) and `[*]` (traversal) child entries, then recurse into items schema
- For primitive types (string, number, boolean, integer): mark as leaf
- Limit recursion depth to 5 levels to prevent infinite schemas with $ref
- Handle `$ref` by logging a warning and stopping recursion (don't resolve $ref — too complex for picker)

**2. Tree rendering in VariablePicker:**

Currently VariablePicker shows a flat list of upstream node outputs. Extend it:

- For each upstream node, check if its config is `model_call` and has:
  - `jsonSchema` (node-level) — show tree under each model output
  - `namedOutputs` with per-artifact `jsonSchema` — show tree under each named output
- When jsonSchema exists: render an expandable tree using the `buildSchemaTree` output
  - Each tree node shows: icon (folder for object/array, leaf icon for primitives), property name, type badge
  - Clicking a leaf node calls `onSelect` with the full path: `{{nodeId.segmentKey.fieldPath}}`
  - Clicking an array node shows both `[0]` and `[*]` options as children
  - Tree nodes are collapsible (click to expand/collapse)
  - Use indentation (padding-left) to show hierarchy (16px per level)
- When no jsonSchema: show flat output reference only (existing behavior, no tree)

**3. Variable insertion format:**

When user clicks a leaf in the tree:
- If it's a direct model output field: `{{nodeId.modelId.fieldPath}}`
- If it's a named output field: `{{nodeId.artifactId.fieldPath}}`
- Build the `VariableRef` with appropriate `fieldPath` value
- Example: clicking "name" under items[0] in node n3's "clause_list" artifact generates `{{n3.clause_list.items[0].name}}`

**4. Visual design:**
- Tree items use smaller font size than top-level outputs
- Expandable nodes show a chevron (right when collapsed, down when expanded)
- Leaf nodes show a variable icon (e.g. `{x}`)
- Type badges: small colored pills — "string" green, "number" blue, "array" purple, "object" gray
- Hover highlight on clickable items
