1. In `packages/shared/src/types.ts`, update the `Model` interface to add three optional parameter fields and an optional providerName field:
   ```typescript
   export interface Model extends BaseEntity {
     providerId: string;
     modelId: string;
     displayName: string;
     isActive: boolean;
     isProviderDisabled: boolean;
     temperature?: number | null;
     maxTokens?: number | null;
     topP?: number | null;
     providerName?: string;
   }
   ```
   Use `?` (optional) because not all API responses include these fields. The parameter fields use `number | null` to match backend schema (null = use API default).

2. In `packages/backend/src/modules/models/models.service.ts`:
   a. Add `providerName: string | null` to the `ModelRow` type.
   b. Update `listActiveModels()` to use a LEFT JOIN on `providers` table to include `providerName`:
      ```typescript
      export async function listActiveModels() {
        return db
          .select({
            ...modelColumns,
            providerName: providers.name,
          })
          .from(models)
          .leftJoin(providers, eq(models.providerId, providers.id))
          .where(eq(models.isActive, true))
          .orderBy(desc(models.createdAt));
      }
      ```
   c. Update the return type to include `providerName`. LEFT JOIN makes `providerName` nullable (`string | null`), which is correct since a model could theoretically reference a deleted provider.
   d. Do NOT modify `listModelsByProvider`, `createModel`, `updateModel`, or other functions — they don't need providerName since the provider context is already known.

CURRENT (buggy):
```typescript
const result = validateRes.data as { data?: { errors?: ValidationError[] } };
const errors: ValidationError[] = result.data?.errors ?? [];
```

REPLACE WITH:
```typescript
const result = validateRes.data as { valid?: boolean; errors?: ValidationError[] };
const errors: ValidationError[] = result.errors ?? [];
```

The backend `POST /:id/validate` returns `{ valid: boolean, errors: WorkflowValidationError[] }` directly — it does NOT wrap in a `{ data: ... }` envelope. The frontend incorrectly assumed the `{ data: ... }` wrapper pattern used by list endpoints.

Do NOT change any other code in the file. This is a single-site fix.
