**input-transform.service.ts:**

1. `handleFileUpload(documentId: string, nodeExecutionId: string, file: File, userId: string)`:
   - Save file to `getUploadPath(documentId)/${nodeExecutionId}/` directory (create if needed)
   - Call `insertDocumentFile` to index in DB
   - Call `parseUploadedFile` to extract text
   - Return { fileId, originalName, parsedText, mimeType, fileSize }

2. `parseUploadedFile(filePath: string, mimeType: string)`:
   - For v1, implement simple text extraction:
     - `.txt` / `.md`: read as UTF-8 text directly
     - `.pdf`: use `pdf-parse` npm package (install via `bun add pdf-parse`) to extract text
     - `.docx`: use `mammoth` npm package (install via `bun add mammoth`) to extract text
     - Images (`.png`, `.jpg`): return placeholder "[图片文件: {filename}]" (OCR deferred)
     - Audio/Video (`.mp3`, `.mp4`): return placeholder "[音视频文件: {filename}]" (transcription deferred)
   - Return extracted text string

3. `confirmInputTransform(documentId: string, nodeExecutionId: string, formData: Record<string, string>, fileOutputs: Array<{fileId: string, parsedText: string}>, userId: string)`:
   - Merge form field values + file parsed texts into outputData
   - Structure outputData as: `{ fields: Record<string, string>, files: Array<{fileId, name, parsedText}>, confirmedAt: string }`
   - Write combined text output to step directory: `getUploadPath(documentId)/${nodeExecutionId}/output.txt`
   - Update nodeExecution.outputData and nodeExecution.inputData with the raw inputs
   - Return updated nodeExecution

**input-transform.routes.ts:**
Add routes under `/runtime/:documentId/input-transform/:nodeExecutionId`:
- `POST /upload` — multipart file upload (use Elysia's native body parsing with t.File()), calls handleFileUpload
- `POST /confirm` — body: { formData, fileOutputs }, calls confirmInputTransform

Register these routes inside the existing runtimeRoutes group in runtime.routes.ts (use Elysia .group() or import and .use()).
