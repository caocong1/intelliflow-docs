<!-- Backend API endpoints created by Plan 01 -->

Provider endpoints (all require admin auth):
- GET /api/providers — returns Provider[] with masked API keys
- POST /api/providers — body: { name, baseUrl, apiKey?, username?, type? } — returns created Provider
- PATCH /api/providers/:id — body: { name?, baseUrl?, apiKey? } — returns updated Provider
- DELETE /api/providers/:id — blocks if has models, returns error
- PATCH /api/providers/:id/status — toggles isActive with cascade to models
- POST /api/providers/:id/test — body: { modelId? } — returns { success, message, latencyMs }

Model endpoints (all require admin auth):
- GET /api/models/by-provider/:providerId — returns Model[]
- POST /api/models — body: { providerId, modelId, displayName, deploymentType? }
- PATCH /api/models/:id — body: { modelId?, displayName?, deploymentType? }
- DELETE /api/models/:id
- PATCH /api/models/:id/status — toggles model isActive

**Page structure:**
- Page header: "AI 模型配置" title + "添加供应商" button (top-right)
- Provider cards in a vertical stack (not grid — providers are few, cards are wide)
- Each provider card contains:
  - Card header: provider name, type badge ("OpenAI 兼容" or "OpenCode"), status badge (active/inactive via Badge component)
  - Card subheader: base URL (truncated if long), API key masked display (e.g. "sk-...abc123")
  - Action buttons in header area: "测试连接" (placeholder — wired in Task 2), "编辑", "停用/启用", "删除"
  - Model list inside card as a mini-table or styled list rows:
    - Columns: 模型名称 (displayName), 模型 ID (modelId), 部署类型 (cloud→"云端"/local→"本地"), 状态 (Badge), 操作 (编辑/停用/删除)
    - When provider is disabled, all model rows appear greyed out (opacity-50 or similar) regardless of individual model status
    - A model is shown as "已启用" only when `isActive && !isProviderDisabled`
  - "+ 添加模型" button at bottom of model list

- On mount: fetch all providers via `api.api.providers.get()`
- For each provider: fetch models via `api.api.models["by-provider"][providerId].get()`
- Use SolidJS createSignal for providers list, createSignal per-provider for models (or a single Map signal)
- Refetch after any mutation (create/update/delete/toggle)

**Cascade display:**
- When provider `isActive` is false, all model rows appear greyed out (opacity-50) regardless of individual model `isActive` status
- Status badge logic: model shows "已启用" only when `isActive && !isProviderDisabled`

Follow UI/UX skill guidelines:

- Loading states for data fetching
