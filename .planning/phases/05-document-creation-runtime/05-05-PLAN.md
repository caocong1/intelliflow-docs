**model-call.service.ts:**

1. `resolvePromptTemplate(template: string, documentId: string, nodeExecutions: NodeExecution[], desensitizeRules: DesensitizeRuleDesc[])`:
   - Replace `{{variableName}}` references with actual content from upstream node outputData
   - Variable format: `{nodeLabel}.{outputName}` — look up nodeExecution by nodeLabel, extract output field
   - If desensitize rules exist (from a prior desensitize node), append to prompt: "注意：以下文本中包含已脱敏的占位符，请保留这些占位符不要修改：\n" + rules formatted as "- [PERSON_1]: 人名\n" etc. (NODE-08)
   - Return resolved prompt string

2. `executeModelCall(documentId: string, nodeExecutionId: string, modelIds: string[], resolvedPrompt: string)`:
   - For each modelId in modelIds:
     - Look up model + provider from DB
     - Store initial ModelOutput with status="pending" in nodeExecution.outputData.models[modelId]
   - Return a ReadableStream that multiplexes SSE events from all models:
     - For each model, call provider API in parallel:
       - POST `${provider.baseUrl}/chat/completions` with `{ model: model.modelId, messages: [{ role: "user", content: resolvedPrompt }], stream: true, temperature: model.temperature ?? undefined, max_tokens: model.maxTokens ?? undefined, top_p: model.topP ?? undefined }`
       - Read response as SSE stream (ReadableStream from fetch)
       - Parse each `data: {...}` chunk, extract `choices[0].delta.content`
       - Emit SSEEvent: { type: "delta", modelId, data: deltaContent, timestamp }
       - On stream end: emit { type: "complete", modelId, data: fullContent, timestamp }
       - On error: emit { type: "error", modelId, data: errorMessage, timestamp }
     - After all streams finish, update nodeExecution.outputData with all ModelOutput records
   - Use `Promise.allSettled` to run models in parallel, each streaming independently

3. `retryModelCall(documentId: string, nodeExecutionId: string, modelId: string, resolvedPrompt: string)`:
   - Re-execute single model, same streaming pattern
   - Update only that model's entry in outputData.models
   - Preserve other models' existing outputs (NODE-14)

4. `selectModelOutput(documentId: string, nodeExecutionId: string, selectedModelId: string)`:
   - Set nodeExecution.selectedOutputKey = selectedModelId
   - Copy selected model's content to nodeExecution.outputData.selectedContent for downstream consumption
   - Write selected content to step directory file

**model-call.routes.ts:**
Routes under `/runtime/:documentId/model-call/:nodeExecutionId`:
- `GET /execute` — SSE endpoint. Resolves prompt, starts parallel model execution, returns SSE stream. Use Elysia's response streaming: return a Response with `Content-Type: text/event-stream` and a ReadableStream body.
- `GET /retry/:modelId` — SSE endpoint for single model retry
- `POST /select` — body: { selectedModelId }, calls selectModelOutput
- `GET /status` — returns current outputData.models (polling fallback if SSE reconnects)

Register in runtimeRoutes.
