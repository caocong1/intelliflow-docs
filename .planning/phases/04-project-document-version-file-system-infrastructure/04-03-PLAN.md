**files.service.ts** — Workspace directory management:

```typescript
import { mkdir } from "node:fs/promises";
import { join } from "node:path";

const WORKSPACE_ROOT = process.env.WORKSPACE_ROOT || "./data/workspaces";

export async function createDocumentWorkspace(documentId: string): Promise<void> {
  await mkdir(join(WORKSPACE_ROOT, "uploads", documentId), { recursive: true });
  await mkdir(join(WORKSPACE_ROOT, "exports", documentId), { recursive: true });
  await mkdir(join(WORKSPACE_ROOT, ".mappings", documentId), { recursive: true });
}

export function getUploadPath(documentId: string): string {
  return join(WORKSPACE_ROOT, "uploads", documentId);
}

export function getExportPath(documentId: string): string {
  return join(WORKSPACE_ROOT, "exports", documentId);
}
```

Per CONTEXT.md: no step subdirectories. Creates uploads/{doc-uuid}/, exports/{doc-uuid}/, and .mappings/{doc-uuid}/ (for NODE-07 desensitization mapping local copy per CONTEXT.md storage decision). Text content goes to DB. Archive = DB flag only, no directory move.

**documents.service.ts** — Business logic:

1. `createDocument(userId, body: {projectId, workflowId, title, description?})` — Insert document with defaults (status: draft, visibility: project). Call createDocumentWorkspace(doc.id) after insert. Return the created document. Verify user is a project member before creating.

2. `listDocuments(projectId, userId, isOwner, params: {page, pageSize, search?, status?, sort?})` —
   - Query documents where projectId matches AND isDeleted = false
   - **Visibility filter (DMGT-06):** If isOwner = true, show all. Otherwise: show docs where visibility='project' OR createdBy=userId OR (visibility='specific' AND userId in documentVisibilityMembers).
   - Search: ilike on title
   - Filter by status if provided
   - Sort by createdAt desc by default
   - Join creator displayName
   - Return { data: Document[], total: number }

3. `getDocument(documentId, userId, isOwner)` — Get single document with visibility check. Join workflow name and creator name.

4. `updateDocument(documentId, body: {title?, description?})` — Update basic info. Only document creator should call.

5. `deleteDocument(documentId)` — Soft delete: isDeleted=true, deletedAt=now. Sets isArchived=true on the document (FSYS-04: archive flag, no directory move).

6. `restoreDocument(documentId)` — Undelete: isDeleted=false, deletedAt=null, isArchived=false.

7. `permanentDeleteDocument(documentId)` — Hard delete from DB (owner only from recycle bin). Also delete related documentVisibilityMembers and documentVersions rows.

8. `updateVisibility(documentId, userId, visibility, memberIds?)` — Update visibility field. If "specific", replace documentVisibilityMembers rows. Only document creator can change visibility.

9. `listDeletedDocuments(projectId, page, pageSize)` — List soft-deleted documents for recycle bin. Owner only.

**documents.routes.ts** — Routes under `/documents` with requireAuth:

```
GET    /documents?projectId=&page=&pageSize=&search=&status=  — listDocuments
POST   /documents                                              — createDocument
GET    /documents/:id                                          — getDocument
PATCH  /documents/:id                                          — updateDocument
DELETE /documents/:id                                          — deleteDocument (soft)
POST   /documents/:id/restore                                  — restoreDocument
DELETE /documents/:id/permanent                                — permanentDeleteDocument
PATCH  /documents/:id/visibility                               — updateVisibility (body: visibility, memberIds?)
GET    /documents/deleted?projectId=&page=&pageSize=           — listDeletedDocuments
```

All routes require auth. Permission checks done in handlers:
- createDocument: check isProjectMember
- listDocuments: check isProjectMember, pass isProjectOwner for visibility bypass
- getDocument: check isProjectMember + visibility filter
- updateDocument: check document creator
- deleteDocument: check document creator OR project owner
- restore/permanentDelete: check project owner
- updateVisibility: check document creator
- listDeletedDocuments: check project owner

Register documentMgmtRoutes in packages/backend/src/index.ts with `.use(documentMgmtRoutes)`.

NOTE: Use `documentMgmtRoutes` as the export name (not `documentRoutes`) to avoid collision with existing documentTypeRoutes.
