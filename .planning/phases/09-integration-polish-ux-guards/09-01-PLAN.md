**DocumentTypeManagement.tsx — Extend delete modal with association pre-check:**

1. Add two new signals:
   - `const [deleteCheckLoading, setDeleteCheckLoading] = createSignal(false);`
   - `const [associatedWorkflows, setAssociatedWorkflows] = createSignal<{id: string; name: string}[]>([]);`

2. When user clicks delete (where `setConfirmAction({ docType, action: "delete" })` is called), immediately after setting confirmAction, trigger the association check:
   - Set `setDeleteCheckLoading(true)` and `setAssociatedWorkflows([])`
   - Call `api.api["document-types"]({ id: docType.id }).associations.get()` via Eden Treaty
   - On success: `setAssociatedWorkflows(result.workflows)` and `setDeleteCheckLoading(false)`
   - On error: `setDeleteCheckLoading(false)` (proceed with standard delete confirm as fallback)

3. Modify the confirm modal content for `action === "delete"`:
   - When `deleteCheckLoading()` is true: show a small spinner with text "正在检查关联..."
