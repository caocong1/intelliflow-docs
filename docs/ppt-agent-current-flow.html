当前 PPT Agent 流程

当前实现已经不是简单的“生成 DeckPlan 然后渲染”。现在流程加入了大纲规划、风格 DNA、逐页 compose、跨页 reviewer 和定向修复。 MiniMax 调用次数变多，但大部分新增调用是单页短上下文。

4 类

MiniMax 文本调用：DeckPlan、单页 compose、整 deck rewrite、整 deck review。

2 次

每页 compose 最多尝试次数：COMPOSE_MAX_ATTEMPTS = 2。

6 项

Deck Reviewer 定向修复最多执行：MAX_TARGETED_FIXES = 6。

当前版本的关键变化

  • 新增 outline_planner：本地生成章节和叙事大纲，但暂时主要用于 trace，不直接进 MiniMax prompt。
  • 新增 style_director：本地抽取 Style DNA，后续每页 compose 都带这个摘要。
  • 新增 slide_composer：逐页调用 MiniMax，单页短上下文，但调用次数可能最多是页数 × 2。
  • 新增 deck_reviewer：本地找跨页问题，最多触发 6 个单页定向修复，也可能触发完整 deck review。
  • DeckPlan 校验前现在会做容错修复，避免模型返回空块、空 prompt、半截 chart/timeline 时直接进入保底。

关于“上下文是否太长”

当前最容易变长的是 rewriteDeckPlan 和 reviewDeck，因为它们会发送完整 DeckPlan。 新增的 composeSlide 只发送单页 JSON 和 Style DNA，单次上下文短，但调用次数多。图片调用仍然很短。
