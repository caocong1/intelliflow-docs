# PPT Scene Single-Slide Prompt: TOC

Model:

`gemini-3.1-pro-preview`

Goal:

生成一页高质量目录页，输出完整 `ppt_scene/v1` JSON，但 `slides` 数组中只能有 1 页。

Prompt:

```text
基于已提供的 `ppt_scene/v1` 协议，输出一个完全合法的 JSON。

任务：
- 只生成 1 页
- 页类型：toc
- 主题：无线网络建设全流程指南
- 语言：zh-CN

目录内容：
- 01 核心差异：无线 vs 有线
- 02 技术演进与建设流程
- 03 场景选型与品牌判断
- 04 运维优化与实施建议
- 05 结论与关键原则

质量目标：
- 必须像“目录海报 / 目录拼贴页”
- 不能像普通章节列表
- 不能像项目符号清单
- 必须有节奏、秩序和视觉导航感

视觉方向：
- editorial
- mosaic
- premium
- navigation poster
- asymmetric

强制要求：
1. 只输出一个完整的 `ppt_scene/v1` JSON。
2. `slides` 中必须且只能有 1 页。
3. 禁止只用一个大文本框列出 5 条目录。
4. 必须使用 3-6 个不同大小的导航容器或模块。
5. 必须有明确编号层级。
6. 目录项之间必须有视觉分区，不允许纯文本堆叠。
7. 必须包含至少一个大尺度背景结构 shape。
8. 必须像“导航页”，不是“内容页”。
9. 可以使用线条、编号块、框体、拼贴块，但不要平均分配。

禁止事项：
- 不要 bullet list
- 不要表格
- 不要咨询模板目录
- 不要五张等大卡片
- 不要后台菜单感

只输出 JSON，不要 Markdown，不要解释。
```
