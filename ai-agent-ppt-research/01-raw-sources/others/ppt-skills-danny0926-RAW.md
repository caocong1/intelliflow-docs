## README.md (full)

```markdown
# Excalidraw Slides

AI-powered hand-drawn presentation generator. Turns text outlines into visually rich, sketch-style PPTX files with editable text.

## How It Works

```
Text / Outline / PDF
  → Phase 1: Content structuring (1 slide = 1 message)
  → Phase 2: Style preset selection (6 themes)
  → Phase 3: Visual-first layout planning (15+ layout types)
  → Phase 4: HTML/CSS + rough.js generation per slide
  → Phase 5: Playwright → PNG rendering + visual review
  → Phase 6: Dual-layer PPTX assembly
```

### Dual-Layer Architecture

Each slide has two layers:
- Layer 1 (background): Hand-drawn decorations rendered as PNG via rough.js + rough-notation
- Layer 2 (foreground): Native python-pptx TextBox / Image elements — fully editable in PowerPoint

## Features

- Visual-first design — 60/40 visual-to-text ratio, max 8-word headlines, tinted image placeholders
- 15+ layout types — split-visual, visual-hero, comparison, big-number, process-flow, icon-grid, timeline, and more
- 6 style presets — Clean Sketch, Bold Marker, Notebook, Blackboard, Blueprint, Watercolor
- Rough.js concept illustrations — 10 reusable patterns (funnel, cycle, Venn, tree, growth arrow, etc.)
- CJK support — automatic Chinese/Japanese/Korean font switching
- Layout variety enforcement — no consecutive repeats, decision tree for visual-first layout selection

## Prerequisites

pip install jinja2 playwright python-pptx lxml
playwright install chromium
```

## 6-Phase Pipeline

```
Input (text / outline / PDF / markdown)
    ↓
Phase 1 — Content Structuring (split into 1-message-per-slide outline)
    ↓
Phase 2 — Style Selection (pick a style preset from STYLE_PRESETS.md)
    ↓
Phase 3 — Layout Planning (assign layout type per slide, enforce variety)
    ↓
Phase 4 — HTML Generation (compose HTML/CSS/JS per slide with DESIGN_SYSTEM.md)
    ↓
Phase 5 — Render + Visual Review (render_slides.py → read PNGs → check → fix)
    ↓
Phase 6 — Assemble (gen_pptx.py → .pptx)
```

## Phase 1: Content Structuring

Break input into a slide outline. Each slide = one single message.

Rules:
- Max 3-5 bullets per slide — prefer visual over text
- Use big numbers for impactful statistics instead of burying in bullets
- Use quotes for memorable phrases
- Insert section breaks for visual breathing room between topics
- If input is a PDF, read it first with the Read tool
- Ask the user for preferred language (English/Chinese) if not clear

### Visual Density Rules

- 60/40 rule: At least 60% of each content slide's area must be visual (images, diagrams, rough.js illustrations, big numbers, charts). Max 40% text.
- Headlines ≤ 8 words: Force concise headings. Details go to speaker notes or supporting text, not slide headings.
- Max 3 bullets: If more than 3 points, split into multiple slides or use icon-grid/visual layout instead.
- Every content slide must have a visual: Either a concept illustration (rough.js), a placeholder image, a diagram, a chart, or a big-number display. No text-only slides except section-break/closing.

## Phase 2: Style Selection

| Preset | Best For |
|--------|----------|
| Clean Sketch (default) | Formal reports, paper reviews, corporate |
| Bold Marker | Keynotes, pitch decks, conference talks |
| Notebook | Reading notes, study materials, tutorials |
| Blackboard | Tech talks, live demos, developer audiences |
| Blueprint | Architecture diagrams, system design |
| Watercolor | Brand presentations, creative proposals |

## Phase 3: Layout Planning

Before generating any HTML, plan layout types across ALL slides to enforce variety.

Available layout types (reference patterns in DESIGN_SYSTEM.md):
- centered-hero — Title slide, centered
- title-bullets — Title + bullet list
- two-column — Two-column grid/split
- three-column — Three-column cards/grid
- big-number — Giant statistic + caption
- quote — Large quote block
- timeline — Horizontal timeline
- process-flow — Step boxes + arrows
- icon-grid — Emoji/icon grid (3-4 columns)
- split-visual — 50/50 or 60/40 image + text side-by-side
- visual-hero — Large rough.js concept illustration (center) + small text caption
- comparison — Side-by-side comparison with visual indicators
- annotated-diagram — Rough.js diagram with labeled callouts
- full-bleed — Giant centered text statement
- section-break — Section divider
- closing — Thank you / Q&A
- diagram — Mermaid.js diagram
- chart — Chart.js visualization
- custom — Freeform layout (for unique one-offs)

### Layout Selection Decision Tree

```
Has a key statistic?          → big-number
Has a quote?                  → quote
Has a process/steps?          → process-flow or timeline
Has comparison data?          → split-visual or comparison
Has a concept to explain?     → visual-hero (rough.js illustration + text)
Has a list of items (≤6)?     → icon-grid
Has a list of items (>6)?     → two-column or three-column with icons
Has a diagram/architecture?   → diagram (image_only) or annotated-diagram
Has an image/photo?           → split-visual (image + text side-by-side)
Default                       → split-visual (placeholder image + text)
```

The goal is visual-first: always choose the most visual layout that fits the content. Text-heavy layouts (title-bullets) should be the last resort, not the default.

Variety rules:
1. Never use the same layout type on consecutive slides
2. title-bullets may appear at most 2× in a 10-slide deck
3. At least 3 different layout types in any 5-slide window
4. Use at least 1 visual slide (big-number, quote, diagram, chart, icon-grid) per 4 content slides
5. Vary rough.js decoration patterns — don't repeat the same decoration on consecutive slides

## Phase 4: HTML Generation

### JSON Schema

```json
{
  "style": {
    "bg": "#fffdf7", "text": "#1a1a2e", "text_muted": "#495057",
    "roughness": 1.0, "stroke_width": 1.5,
    "font_map": {
      "heading": "Ink Free",
      "body": "Ink Free",
      "annotation": "Ink Free",
      "cjk": "Microsoft JhengHei"
    }
  },
  "slides": [
    {
      "id": 1,
      "render_mode": "dual",
      "html": "<div>...your layout HTML...</div>",
      "css": ".your-class { ... }",
      "shapes_js": "drawRect(30, 30, 1860, 1020, {...});",
      "annotations_js": "annotateElement('el-id', {type:'underline'});",
      "accent_index": 0,
      "elements": [
        { "type": "textbox", "x": 160, "y": 350, "w": 1600, "h": 200,
          "paragraphs": [ { "text": "Title", "font": "heading", "size": 54, "color": "#1864ab", "bold": true, "align": "center" } ] },
        { "type": "image", "x": 900, "y": 200, "w": 800, "h": 500,
          "src": null, "label": "[Insert photo here]" }
      ]
    }
  ]
}
```

### Slide Fields

| Field | Required | Description |
|-------|----------|-------------|
| id | Yes | Slide number (1-based) |
| render_mode | No | "dual" (default) — bg PNG + native elements. "image_only" — full-bleed PNG (for Mermaid/Chart.js slides) |
| html | Yes | HTML content for the slide body (used for rendering + annotation targets) |
| css | No | Additional CSS rules (injected into <style>) |
| shapes_js | No | rough.js decoration calls (runs after fonts load) |
| annotations_js | No | rough-notation calls (runs after fonts load) |
| accent_index | No | Force accent color (0-5). Auto-rotates by slide index if omitted |
| elements | No | Array of native PPTX elements (textbox/image). Only used when render_mode is "dual" |

### Element schema (textbox)
```json
{
  "type": "textbox",
  "x": 160, "y": 100, "w": 1600, "h": 200,
  "paragraphs": [
    { "text": "Heading text", "font": "heading", "size": 54, "color": "#1864ab", "bold": true,
      "italic": false, "align": "center", "bullet": false, "level": 0,
      "space_before": 0, "space_after": 8,
      "runs": [
        { "text": "Bold part", "bold": true, "font": "heading", "size": 54, "color": "#1864ab" },
        { "text": " normal part", "font": "body", "size": 32 }
      ] }
  ]
}
```

### Element schema (image)
```json
{
  "type": "image",
  "x": 900, "y": 200, "w": 800, "h": 500,
  "src": "path/to/image.png",
  "label": "[Insert experiment screenshot]",
  "bg_color": "#d0ebff",
  "bg_opacity": 0.3,
  "border_color": "#1971c2",
  "icon": "📷",
  "label_color": "#495057"
}
```

If src is null or file doesn't exist, creates a placeholder rectangle with label text.

### Image Placeholder Label as Prompt

When src is null, the label text appears inside the placeholder box. Write the label as a ready-to-use image generation prompt so the user can copy-paste it directly into any AI image tool (Gemini, DALL-E, Midjourney, etc.).

Rules:
1. Describe the visual, not the concept — "A funnel diagram with 3 layers narrowing from top to bottom" not "[Insert funnel diagram]"
2. Include style cues — add "sketch style", "hand-drawn", "white background", "minimal flat illustration"
3. Be specific — mention colors, composition, key elements
4. Keep under 30 words — prompt must fit legibly inside the placeholder box
5. Match slide context — the prompt should describe an image that reinforces the slide's message

Bad: "[Insert diagram here]", "[Add relevant image]", "[Experiment results]"
Good: "Hand-drawn funnel: raw data → filtering → clean insights, sketch style, white background"
Good: "Minimal illustration of a neural network with 3 layers, hand-drawn style, blue and orange nodes"
Good: "Before/after comparison: messy desk vs organized desk, sketch style, split view"

### Design Principles

1. 1 slide = 1 message
2. Never repeat layouts: Vary structure across slides
3. Typography contrast: Headings 64-96px (Caveat), body 28-36px (Patrick Hand)
4. Max 3-5 bullets per slide — prefer visual over text
5. Vary decorations: Different rough.js patterns per slide
6. Whitespace is design: Don't fill every pixel
7. Accent colors rotate: Let the palette cycle naturally
8. Follow style preset: Respect chosen preset's roughness/strokeWidth

## Phase 5: Render + Visual Review

```bash
python .claude/skills/excalidraw-slides/render_slides.py \
  --deck deck.json \
  --outdir slides_tmp/ \
  --mode both
```

After rendering, read each slide_XX.png (full preview) with the Read tool to visually check for:
- Text overflow or clipping
- Overlapping elements
- Empty/blank areas that should have content
- Decoration covering text (z-index issues)
- Font rendering issues

If any issues are found, fix the slide JSON and re-render only the affected slides.

## Phase 6: Assemble PPTX

Dual-layer mode (editable text + hand-drawn background):
```bash
python .claude/skills/excalidraw-slides/gen_pptx.py \
  --deck deck.json \
  --imgdir slides_tmp/ \
  --output "Presentation.pptx"
```

### Choosing render_mode per slide

- Use "dual" (default) for slides with text/bullet content — user can edit text in PowerPoint
- Use "image_only" for Mermaid diagrams, Chart.js charts, or complex visual slides — these are baked into the PNG since they can't be reproduced natively in PPTX

## Tips

- For CJK text, fonts include Noto Sans TC — works automatically
- Slides with <pre class="mermaid"> or new Chart() need ~1500ms render time
- Canvas is 1920x1080 with 80px padding → usable area is 1760x920
- 1 px = 6350 EMU (for gen_pptx.py conversion)

| Layout Type | Primary Decoration | Secondary | Avoid |
|------------|-------------------|-----------|-------|

| big-number | Circle/ellipse around number | Underline on caption | Rectangles |
| title-bullets | Corner brackets | Bullet accent dots | Full frames |
| icon-grid | Card outlines per item | None needed | Connecting lines |
| comparison | Box outlines per column | "vs" divider line | Decorations inside boxes |
| annotated-diagram | Diagram IS the decoration | Callout lines to labels | Border frames |

| process-flow | Step boxes + connecting arrows | Number circles | Background fills |
| quote | Large quotation marks (drawn) | Subtle underline | Box frames |
| two-column | Vertical divider line | Corner accents | Double borders |
| section-break | Flanking horizontal lines | None needed | Full frames |
| closing | Decorative border frame | Corner accents | Heavy illustrations |
| full-bleed-statement | Underline on key phrase | Corner accents | Full borders |

## render_slides.py (key behaviors)

- Uses Jinja2 + Playwright (chromium) headless.
- Three modes: `full` (default), `bg_only` (transparent text via Ghost CSS), `both`.
- Ghost CSS used in bg_only: makes `#content` and all children transparent, hides images/svg/canvas — but keeps the rough.js `<svg id="rough-layer">` visible. This is critical for dual-layer: it lets us render just the background to PNG while annotations_js targets still exist at the correct positions.
- Accent color rotation by slide index using 6-color array (matches design system).
- Mermaid/Chart.js slides get ~1500ms extra render time.

## gen_pptx.py (key behaviors)

- 16:9 slide dimensions: 13.333 × 7.5 inches (12192000 EMU wide).
- Coordinate conversion: `1 px = 6350 EMU` (PX_TO_EMU = SLIDE_WIDTH / 1920).
- Default font map: heading/body/annotation = "Ink Free" (Windows hand-drawn font); cjk = "Microsoft JhengHei".
- Dual-layer assembly:
  1. Insert background PNG (bg_only render) as full-bleed picture.
  2. Place native python-pptx TextBox / Image shapes on top using `elements[]` coordinates.
  3. CJK text auto-handled via `a:ea` typeface on every run.
- Image element behavior: if `src` is a real file, insert image; if null, draw a rectangle with `bg_color` fill, optional `icon` emoji, and `label` text as a copy-paste-ready prompt.
- Image-only mode: full-bleed PNG insertion (legacy / for diagrams).
