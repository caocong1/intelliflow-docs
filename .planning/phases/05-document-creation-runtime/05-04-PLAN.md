**desensitize.service.ts:**

1. `detectSensitiveInfo(text: string, localModelId: string | null, ruleTypes: string[], placeholderFormat: string)`:
   - If localModelId is provided:
     - Look up model + provider from DB (models JOIN providers)
     - Call provider's OpenAI-compatible API: POST `${provider.baseUrl}/chat/completions` with:
       - Authorization: `Bearer ${provider.apiKey}`
       - Body: `{ model: model.modelId, messages: [{ role: "system", content: systemPrompt }, { role: "user", content: text }], temperature: 0.1 }`
       - systemPrompt instructs model to identify sensitive info types from ruleTypes, return JSON array: `[{ "original": "张三", "type": "person_name", "description": "人名" }]`
     - Parse model response JSON
   - If no localModelId: use simple regex-based fallback detection for common patterns (phone numbers, email addresses, ID numbers)
   - Generate placeholders using placeholderFormat (e.g., "[PERSON_1]", "[PHONE_1]") with incrementing counters per type
   - Return: `Array<{ original: string, placeholder: string, sensitiveType: string, description: string, startIndex: number, endIndex: number }>`

2. `confirmDesensitization(documentId: string, nodeExecutionId: string, items: Array<{ original: string, placeholder: string, sensitiveType: string }>, sanitizedText: string, userId: string)`:
   - Insert confirmed items into desensitizeMappings table
   - Store sanitizedText (text with placeholders replacing originals) as nodeExecution.outputData
   - Also write sanitizedText to step directory file
   - Also write a local mapping copy to `.mappings/{documentId}/{nodeExecutionId}.json` (per NODE-07)
   - Return updated nodeExecution

3. `getDesensitizeRules(documentId: string, nodeExecutionId: string)`:
   - Query desensitizeMappings for this execution
   - Return DesensitizeRuleDesc[] — ONLY placeholder + sensitiveType + description (NO originalValue)
   - This is what gets injected into model call prompts (NODE-08)

**desensitize.routes.ts:**
Routes under `/runtime/:documentId/desensitize/:nodeExecutionId`:
- `POST /detect` — body: { text }, calls detectSensitiveInfo using the node's config (localModelId, ruleTypes, placeholderFormat from workflow)
- `POST /confirm` — body: { items, sanitizedText }, calls confirmDesensitization
- `GET /rules` — returns getDesensitizeRules (used by model call executor in Plan 05)

Register in runtimeRoutes.
