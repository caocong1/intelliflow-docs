**packages/frontend/src/components/workspace/nodes/ModelCallExecutor.tsx:**

Extend to handle three new scenarios:

**A. format_error status display:**
- When a model output has `status === "format_error"`:
  - Show a red error box at the top of the model output card with red border
  - Display each error from `formatErrors` array as a bullet point (e.g. "JSON 语法错误: Unexpected token at line 5")
  - Card border turns red (`border-red-500`)
  - Below the error box, show the content in an editable textarea (monospace font) for manual editing
  - Add "重新验证" button that calls POST `.../revalidate` with the current edited content
  - Add "AI 修复" button next to revalidate:
    - Calls POST `.../ai-fix` endpoint via SSE
    - Shows streaming response in a side-by-side or diff view
    - After stream completes, user sees "采用修复" and "取消" buttons
    - "采用修复" replaces current content and auto-triggers revalidate
  - On successful revalidation (status becomes "completed"), remove error box and show normal output

**B. Named output cards rendering:**
- When `outputData.namedOutputs` exists and is not empty:
  - If multi-model: group by model first (show model name as section header, e.g. "GPT-4o"), then render NamedOutputCard list within each group
  - If single model: render NamedOutputCard list directly
  - Each card allows independent editing via `onContentChange` that updates `outputData.namedOutputs[id].content`
  - Save individual artifact content via existing node output update mechanism (PATCH style)

**C. Fallback warning:**
- When `outputData.fallbackWarning === true`:
  - Show a yellow warning bar at the top: "模型未按预期格式输出，已合并为单个产物"
  - Below the warning, render a single NamedOutputCard with id="_default"
  - Add a "重试" button that triggers re-generation

**Integration notes:**
- Check `outputData.namedOutputs` to decide between named output card mode vs existing single output mode
- Preserve all existing single-output behavior when namedOutputs is not present
- The model comparison view (if multi-model) should align named output cards by artifact name across models
