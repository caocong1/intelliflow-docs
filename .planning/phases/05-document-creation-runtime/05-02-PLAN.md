Create `packages/backend/src/modules/runtime/` directory with two files:

**runtime.service.ts** — Core runtime orchestration logic:

1. `initDocumentExecution(documentId: string, userId: string)`:
   - Load document + workflow (via getDocument + db query on workflows table)
   - If nodeExecutions already exist for this document, return existing state (resume)
   - Otherwise: topologically sort workflow nodes using edges (BFS from root nodes — nodes with no incoming edges), create one nodeExecution row per node with stepOrder, status="pending" for all, set first node to "in_progress"
   - Update document status to "in_progress" if currently "draft"
   - Return DocumentRuntimeState

2. `getDocumentRuntimeState(documentId: string)`:
   - Query all nodeExecutions for document ordered by stepOrder
   - Get workflow name from documents JOIN workflows
   - Compute currentNodeIndex (first node that is "in_progress", or last completed + 1)
   - Return DocumentRuntimeState

3. `advanceNode(documentId: string, nodeExecutionId: string, userId: string)`:
   - Set current node status to "completed", completedAt = now
   - Create version snapshot via existing createVersionSnapshot (from versions.service)
   - Find next pending node by stepOrder, set to "in_progress", startedAt = now
   - If next node's config has autoAdvance=true and nodeType is "restore", auto-complete it (restore nodes can auto-execute)
   - If no next node, set document status to "completed"
   - Return updated DocumentRuntimeState

4. `rollbackToNode(documentId: string, targetStepOrder: number, userId: string)`:
   - Per user decision: preserve downstream outputs but mark them as "pending" (needs re-execution)
   - Set target node to "in_progress"
   - Set all nodes after target to "pending" (reset startedAt, completedAt to null, keep outputData)
   - Return updated DocumentRuntimeState

5. `saveNodeDraft(documentId: string, nodeExecutionId: string, data: Record<string, unknown>)`:
   - Update nodeExecution.outputData with provided data
   - Update nodeExecution.updatedAt
   - This enables auto-save on every modification (RECV-01)

6. `skipNode(documentId: string, nodeExecutionId: string, userId: string)`:
   - Verify node config has skippable=true (load workflow, find matching node config)
   - Set node status to "skipped", advance to next node
   - Return updated DocumentRuntimeState

**runtime.routes.ts** — REST endpoints using requireAuth:

- `POST /runtime/:documentId/init` — calls initDocumentExecution, returns state
- `GET /runtime/:documentId` — calls getDocumentRuntimeState
- `POST /runtime/:documentId/advance/:nodeExecutionId` — calls advanceNode
- `POST /runtime/:documentId/rollback` — body: { targetStepOrder }, calls rollbackToNode
- `PUT /runtime/:documentId/nodes/:nodeExecutionId/draft` — body: { data }, calls saveNodeDraft
- `POST /runtime/:documentId/skip/:nodeExecutionId` — calls skipNode

All routes check project membership (reuse isDocumentProjectMember from versions.service).

**index.ts** — Register `runtimeRoutes` in the Elysia app chain.
