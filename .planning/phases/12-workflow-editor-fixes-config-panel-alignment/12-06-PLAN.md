**1. Edge flow animation** — Add CSS animation to EdgeRenderer:
- Add to the frontend's global CSS or a scoped stylesheet:
  ```css
  @keyframes edge-flow {
    from { stroke-dashoffset: 24; }
    to { stroke-dashoffset: 0; }
  }
  .edge-animated {
    stroke-dasharray: 8 4;
    animation: edge-flow 0.6s linear infinite;
  }
  ```
- Apply `class="edge-animated"` to the visible path in EdgeRenderer
- Respect `prefers-reduced-motion`: wrap animation in `@media (prefers-reduced-motion: no-preference)` or check via JS
- Selected edges: thicker stroke (3px), brighter indigo (#4f46e5), and pulsing glow effect
- Support line type switching: add a small control in the toolbar or per-edge context to switch between bezier/straight/step. Default: bezier.

**1b. Edge midpoint drag interaction** — Allow reshaping edges by dragging their midpoint:

In `EdgeRenderer.tsx`:
- Render a draggable circle (r=6, `fill: white, stroke: #6366f1, stroke-width: 2`) at the edge midpoint (`labelX, labelY` from the path result). Only show on hover or when edge is selected.
- On `onMouseDown` of the midpoint circle: start drag. Set `pointer-events: all` on the circle. Attach window `mousemove` and `mouseup` listeners.
- On `mousemove`: convert mouse position to flow coordinates, compute new control points. For bezier edges, override the two control points symmetrically around the midpoint drag position. For step edges, adjust the middle segment's X position (horizontal drag).
- On `mouseup`: finalize — update `FlowEdgeData.controlPoints` in the store via a new `updateEdgeControlPoints(edgeId, controlPoints)` helper. Push undo snapshot.
- The circle should have `cursor: grab` (and `cursor: grabbing` during drag).

In `lib/flow-engine/store.ts` (minor addition):
- Add `updateEdgeControlPoints(id: string, controlPoints: Array<{ x: number; y: number }> | undefined)` helper that sets `edges[index].controlPoints`.

In `lib/flow-engine/edge-paths.ts`:
- Update `getBezierPath` to accept optional `controlPoints` parameter. If provided, use those instead of auto-calculated control points. The path becomes `M sx,sy C cp[0].x,cp[0].y cp[1].x,cp[1].y tx,ty`.
- Update `getStepPath` similarly: if `controlPoints[0]` exists, use its `x` as the middle segment X position instead of `(sourceX + targetX) / 2`.

In `FlowCanvas.tsx`:
- When rendering EdgeRenderer, pass the edge's `controlPoints` through to the path calculation.
- Double-click on edge midpoint circle resets `controlPoints` to `undefined` (back to auto-calculated).

**2. Alignment guides** — Create `alignment.ts` and `AlignmentGuides.tsx`:

`alignment.ts` — See RESEARCH.md Pattern 8:
- `computeAlignmentGuides(draggingNode, otherNodes, threshold=5)` returns `{ guides: Guide[], snappedX, snappedY }`
- Checks left-left, left-right, right-left, right-right, center-center for vertical guides (X axis)
- Checks top-top, top-bottom, bottom-top, bottom-bottom, center-center for horizontal guides (Y axis)
- Returns snapped position if within threshold

`AlignmentGuides.tsx` — SVG lines:
- For each guide, render a line spanning a large range (e.g., -10000 to 10000) at the guide position
- Style: `stroke: #6366f1; stroke-width: 1; stroke-dasharray: 4 2; opacity: 0.6`
- Only rendered during active node drag

Integrate into FlowCanvas:
- During node drag, call `computeAlignmentGuides` with the dragging node rect and all other node rects
- If snappedX/snappedY returned, use those for the node position (snap effect)
- Render `<AlignmentGuides guides={activeGuides()} />` in the SVG layer
