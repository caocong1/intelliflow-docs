Create `packages/backend/src/modules/versions/` directory.

**versions.service.ts:**

1. `createVersionSnapshot(documentId, nodeId, nodeLabel, snapshotData: Record<string,unknown>, userId)` —
   - Compute next versionNumber: SELECT MAX(versionNumber) + 1 for this documentId (default 1)
   - Insert into documentVersions
   - Return the created version

2. `listVersions(documentId)` —
   - Select all versions for documentId, ordered by versionNumber DESC
   - Join creator displayName
   - Return DocumentVersion[]

3. `getVersion(versionId)` — Get single version with creator name.

4. `getVersionDiff(versionIdA, versionIdB)` —
   - Load both versions
   - Extract text content fields from snapshotData
   - For each common field, compute line-by-line diff:
     - Split both texts by newline
     - Use simple LCS-based diff: iterate through lines, mark as "added", "removed", or "unchanged"
     - Include line numbers for both sides
   - Return VersionDiffResult

   **Diff algorithm:** Use a simple approach for v1:
   - Extract all string values from snapshotData (the text content of node outputs)
   - For each matching key, split into lines and compute diff
   - Lines only in old = "removed", lines only in new = "added", matching = "unchanged"
   - A basic O(n*m) LCS or even simpler longest-common-subsequence works for document-length texts

**versions.routes.ts** — Routes under `/versions` with requireAuth:

```
POST   /versions                    — createVersionSnapshot (body: documentId, nodeId, nodeLabel, snapshotData)
GET    /versions?documentId=        — listVersions
GET    /versions/:id                — getVersion
GET    /versions/:idA/diff/:idB     — getVersionDiff
```

Permission: verify user is a member of the document's project for all operations.
