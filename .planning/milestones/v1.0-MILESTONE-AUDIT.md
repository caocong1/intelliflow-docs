### Integration Issues Found

**1. DTYPE-04 Document Association Guard — TODO Stub (Medium)**

`document-types.service.ts` lines 134-136 has a `// TODO` comment where a document association query should be. The `documents` table exists since Phase 4 with a `documentTypeId` FK. The workflow guard (Phase 9) works correctly, but deleting a document type with associated documents succeeds silently, potentially orphaning records. The route handler already handles the `"HAS_ASSOCIATED_DOCUMENTS"` error string — only the DB query is missing.

**2. Prompt Optimize Auth Level (Low, Policy)**

`/api/prompts/optimize` uses `requireAuth` instead of `requireAdmin`. Any authenticated non-admin user can trigger AI-backed prompt optimization, which incurs API compute cost. No requirement ID directly maps to this, but it is a security/cost policy concern.
