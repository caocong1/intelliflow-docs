## 1. spec_lock.json layer

**Borrowed from**: hugohe3/ppt-master `skills/ppt-master/SKILL.md` rule #8.

**Verbatim quote**:
> "SPEC_LOCK RE-READ PER PAGE — Before generating each SVG page, MUST `read_file <project_path>/spec_lock.md`. Colors / fonts / icons / images MUST come from this file — no memory values, no on-the-fly invention."

**Why it works at PPT Master**:
- Author Hugo He is a Consulting Engineer; spec_lock mirrors how consulting templates lock brand values
- The rule is enforced by `update_spec.py` (only color + typography are propagable; everything else is immutable)
- `svg_quality_checker.py` enforces drift detection (errors block, no auto-fix)

**Deeper insight from RAW capture** (`01-raw-sources/others/ppt-master-RAW.md` lines 162-163):
> "the script-generation path was tried on a feature branch and abandoned: cross-page visual consistency depends on per-page authoring with full upstream context, which a generator script cannot reproduce."

**Implication for IntelliFlow**: don't refactor Layer 4 into "batch generate all pages in parallel" optimization — sequential, contextually anchored generation is essential for coherence.

**Also borrowed from**: arcsin1/oh-my-ppt's `DesignContract` (similar concept, JSON-based, generated by `buildDesignContractSystemPrompt`).

**Skipped**: PPT Master's full SVG-as-source-of-truth pipeline (overkill for IntelliFlow's current HTML middle layer).

## 2. NEVER-list density

**Borrowed from**:
- anthropics/skills/pptx — 8 explicit "Don't ..." rules ("NEVER use accent lines under titles", "Don't default to blue", ...)
- AionUi morph-ppt-3d SKILL.md — same accent-line ban + H4/H6/H7 hard rules
- design-taste-frontend (leonxlnx) — LILA BAN (purple-to-blue AI gradients) + monospace numbers rule

**Why density matters**: anthropics/pptx authors explicitly call out — "These are hallmarks of AI-generated slides." LLMs default to producing them; only explicit NEVER bans counter the default.

**Deeper insight**: the NEVER-list works in proportion to its specificity. Vague bans ("avoid generic aesthetics") have low effect. Concrete bans ("no centered body text", "no accent lines under titles") have high effect because the LLM cannot dispute them.

**Pattern**: every borrowed item has a **specific behavior it counters**. Generic "be tasteful" guidance has zero effect on LLM behavior — IntelliFlow's current `SYSTEM_DESIGN` has 4 generic items, hence weak.

## 3. Visual QA subagent loop

**Borrowed from**:
- anthropics/skills/pptx — most explicit subagent QA spec
- icip-cas/PPTAgent — REPL feedback for editing actions (academic version)
- daymade/claude-code-skills/ppt-creator — 10-dim × 10-point RUBRIC with ≥75 ship threshold
- impeccable/critique — double-blind Assessment A (LLM) + B (detector) pattern

**Verbatim mindset quote** (anthropics):
> "Assume there are problems. Your job is to find them. Your first render is almost never correct. Approach QA as a bug hunt, not a confirmation step. If you found zero issues on first inspection, you weren't looking hard enough."

**Why it works**: separates "generation" from "review" — the generator LLM has authoring bias ("looks fine, I wrote it"); a fresh subagent has no such bias.

**Deeper insight from PPTAgent**: their PPTEval rubric is **published** (Content / Design / Coherence, 1-5 scale, anchored rubrics, κ ≈ 0.59 inter-annotator agreement). Use as reference for our rubric.

**Caveat from impeccable/critique**: "Assessment Independence: Design review (Assessment A) completes first without seeing detector output (Assessment B). Both must finish before synthesis." — to prevent detector numbers from anchoring the LLM review.

**Adapted for IntelliFlow**: skip PPTAgent's REPL editing-action feedback loop (different architecture). Borrow only the **scoring + threshold + iteration cap** pattern from daymade.

## 4. Three-dimensional image lock

**Borrowed from**: hugohe3/ppt-master Strategist's 8th confirmation.

**Verbatim quote**:
> "Strategist locks three orthogonal dimensions for AI-generated images:
> 1. `rendering` — visual style family (vector-illustration / editorial / 3d-isometric / sketch-notes / …)
> 2. `palette` — how the deck's HEX values are *used* (proportion + role + temperament)
> 3. `type` — per-image internal composition (background / hero / framework / comparison / …)
>
> Items 1-2 are deck-wide and written into `spec_lock.md`. Image_Generator assembles every per-image prompt from the locked rendering + palette + per-image type. **Without this, every image gets its own style drift and the deck reads as a stack of unrelated illustrations.**"

**Why it works**: the LLM generates image prompts under three orthogonal constraints — uniform rendering style guarantees visual family; uniform palette usage guarantees color consistency; per-image type allows narrative variety.

**Single most directly applicable insight** to IntelliFlow — our current `imageLanguage: "technical_illustration_plus_real_photo"` is one dimension, ambiguous. Three dimensions are sufficient AND necessary.

## 5. Curated palette pool

**Borrowed from**: anthropics/skills/pptx + iOfficeAI/AionUi morph-ppt-3d (both ship the IDENTICAL 10 palettes).

**Two independent projects converging on the same 10** = evidence this set is well-validated.

**Why it works**: the LLM cannot "invent" a palette in the same brief moment it's also writing slide content. Curated pool provides a stable cognitive anchor.

**Specificity test** (anthropics):
> "If swapping your colors into a completely different presentation would still 'work,' you haven't made specific enough choices."

The 10 palettes each have a CHARACTER (Forest & Moss for sustainability, Cherry Bold for keynotes, etc.). Generic blue/gray would fail this test — they get banned.

**For IntelliFlow**: use as FALLBACK when brief is ambiguous, not as default override of explicit brief. Brief explicit → derive; brief vague → preset.

## 6. Hard rules H1-H10 quantification

**Borrowed from**: danny0926/ppt-skills (60/40 ratio, ≤8 word headlines, ≤3 bullets) + AionUi (H4 ≥16pt, H6 dark contrast, H7 speaker notes) + anthropics (title ≥2× body, 0.5" margin) + daymade (font hierarchy 34/24/18/14).

**Cross-validation**: 5 of 19 analyzed projects independently arrive at body-font-size ≥ 16pt as the **single most violated rule** by LLMs. Strong signal it should be quantified and enforced.

**Insight from danny0926**: "Hard numerical constraints beat soft prompts" — explicitly framed as a design principle, not an accident.

## 7. Visual element checkpoint (every 3 slides, ≥ 1 visual)

**Borrowed from**: iOfficeAI/AionUi morph-ppt-3d.

**Verbatim quote**:
> "Every 3 content slides, at least 1 must contain a non-text visual element: icon in colored circle / colored block / large stat number / chart / gradient background / shape composition. Text-only slides are only allowed for: quotes, code examples, pure tables."

**Why it works**: this is the operationalization of PPTAgent PPTEval's Design 3→4 score boundary (where "supplementary visual elements such as icons, backgrounds, images, or geometric shapes (like rectangles)" tip from "plain" to "harmonious").

**Why "every 3 slides" not "every slide"**: text-only slides serve specific purposes (section breaks, quotes, code). Forcing every slide to be visual would harm content. 1-in-3 ratio is the empirically observed minimum.

## 8. Variance Mandate (no consecutive same-archetype)

**Borrowed from**:
- high-end-visual-design (leonxlnx) — "Variance Mandate"
- danny0926/ppt-skills — "Never use the same layout type on consecutive slides" + "At least 3 different layout types in any 5-slide window"
- impeccable/distill — "layout progression"

**Why it works**: LLMs default to repeating successful patterns. After authoring 3 great two-column slides, the LLM gravitates to four, five, etc. Hard variance check breaks this.

**Implementation**: validate at Layer 3 (PageBrief generation), not Layer 4. By the time Layer 4 sees the brief, archetype is locked.

## 9. Speaker notes (45-60 second each)

**Borrowed from**: daymade/ppt-creator + AionUi H7 + slide-deck-ai narrative arc.

**Structure** (daymade): opening → core assertion → evidence explanation → transition.

**Why 45-60 seconds**: typical PPT delivery pacing. 20-slide deck = 15-20 minute talk → matches Series A pitch / classroom segment / executive briefing rhythm.

**Why mandatory**: forces the LLM to write content COHERENT with what the slide visualizes. If a slide has 3 bullets and the speaker note treats them in different order, that signals incoherence.

## 10. RUBRIC (10×10) with iteration cap

**Borrowed from**: daymade/claude-code-skills/ppt-creator references/RUBRIC.md.

**Verbatim rubric items**:
1. Goal Clarity
2. Story Structure (Pyramid completeness)
3. Slide Assertions (assertion-style headings)
4. Evidence Quality
5. Chart Fit
6. Visual & Accessibility
7. Coherence & Transitions
8. Speakability (45-60s/slide)
9. Deliverables Complete
10. Robustness (gaps marked, fallback plan)

**Threshold ≥ 75 / 100. Max 2 iterations.**

**Why it works**: bounded refinement loop. LLM-as-judge becomes meaningful when scored against anchored rubric items — not just "rate this slide 1-10".

**Adapted for IntelliFlow**: items 1-3, 6, 7, 10 directly transferable. Items 4 (Evidence Quality), 8 (Speakability) require speaker notes. Items 5, 9 are pipeline-level.

## 11. Pyramid Principle + Assertion-Evidence

**Borrowed from**: daymade/ppt-creator + slide-deck-ai initial prompt template.

**Verbatim**:
- "Each slide uses assertion-style headings (complete sentences), with body content providing evidence (charts/tables/diagrams/data points)"
- slide-deck-ai: "Why Most Agile Transformations Fail — And What to Do Instead rather than Agile Transformation"

**Why it works**: forces LLM to commit to a thesis per slide rather than producing topic-tag headlines that don't predict the body content. Topic labels ("Agile Transformation") are AI-default and provide no information density.

**Caveat**: this is a strong stylistic choice. Some decks (e.g. internal training materials, educational) prefer topic-tag style. **Don't force it; surface it as a choice in intake form.**

## 12. Dual-layer editable (background PNG + native text overlay)

**Borrowed from**: danny0926/ppt-skills.

**Mechanism**:
- Render rough.js / decorative HTML to PNG via Playwright with Ghost CSS (makes #content transparent but keeps SVG decorative layer visible)
- Overlay native python-pptx TextBox + Image elements using `1 px = 6350 EMU` conversion
- Result: looks hand-drawn (decoration), but text remains editable in PowerPoint

**Why valuable for IntelliFlow**: bridges "visual fidelity" (image-backed) and "editable" (native) — currently we have separate paths for each.

**Adapted**: IntelliFlow's `html_fidelity` path is conceptually similar. Borrow the Ghost CSS technique to extract just the decorative layer as PNG, overlay editable text via pptxgenjs `addText`.

## 13. Intake form with safe defaults

**Borrowed from**: daymade/ppt-creator `references/INTAKE.md`.

**10 questions** with default fallbacks:
1. Audience (default: general public)
2. Core objective (default: "understand and accept")
3. Desired action/decision (default: agree to next step)
4. Duration & slide count (default: 15-20 min, 12-15 slides)
5. Tone & style (default: professional, clear, friendly)
6. Topic scope (default: given topic + 1 layer related)
7. Must-include / taboos (default: none)
8. Available data/tables (default: none)
9. Brand & visual constraints (default: built-in neutral theme)
10. Deliverable format (default: PPTX if available, else Markdown)

**Why it works**: prevents "the LLM fills with assumptions silently" — instead, either user answers or LLM uses safe default + notes assumption in speaker notes (transparency).

**Adapted for IntelliFlow**: not strictly necessary since IntelliFlow already has structured `VisualBrief` schema. But for fields not in VisualBrief (audience / duration / scope), this intake pattern fills the gap.
