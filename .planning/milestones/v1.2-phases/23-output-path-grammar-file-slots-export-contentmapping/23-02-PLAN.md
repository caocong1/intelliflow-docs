**model-call.service.ts — extract resolveRef():**

Create and export a new `resolveRef()` function with the 6-level priority chain. Follow the exact implementation from 23-RESEARCH.md "resolveRef() -- Extracted from resolvePromptTemplate" section:

```typescript
export function resolveRef(
  ref: { nodeId: string; outputId: string; fieldPath?: string },
  nodeExecs: Array<{ nodeId: string; outputData: Record<string, unknown> | null }>,
): string | undefined
```

Priority chain:
1. `od.fieldsByKey?.[segmentKey]` — machineKey lookup
2. `od.fields?.[segmentKey]` — UUID fallback
3. `od.fileSlots?.[segmentKey]` — file slot (returns `.text`)
4. `od.namedOutputs?.[segmentKey]` — named outputs (Phase 24 stub, returns `.content`)
5. `od.models?.[segmentKey]` — model output (returns `.content`)
6. `od[segmentKey]` — direct property (text, confirmedAt, etc.)

Refactor `resolvePromptTemplate()` to use `resolveRef()` internally. The regex replacement loop (`/\{\{([^}]+)\}\}/g`) stays, but instead of inline field lookup logic, each match calls `resolveRef({ nodeId, outputId: segmentKey }, nodeExecs)`. Keep the desensitize rules append logic unchanged. Preserve the `ResolvedPromptResult` return type and `mapping` tracking.

**input-transform.service.ts — add fieldsByKey and fileSlots:**

Update `confirmInputTransform()` to build two new fields in outputData:

1. `fieldsByKey`: Map machineKey -> value for fields that have machineKey set. To get FormFieldDef configs, the function needs to load the workflow definition. Query: get nodeExecution -> get document -> get workflow -> find matching node in workflow.nodes JSONB -> extract config.formFields. Build `fieldsByKey` by iterating formFields, for each field with `machineKey`, set `fieldsByKey[field.machineKey] = formData[field.id]`.

2. `fileSlots`: Group fileOutputs by their associated fileSlotId. Again use the loaded formFieldDefs to find which file fields have fileSlotId. For each file slot, aggregate: `{ text: concatenated parsedText of files in that slot, files: array of file info }`. The confirm endpoint body already receives `fileOutputs` with `fileId, name, parsedText`. To associate files with slots, extend `ConfirmInputTransformParams.fileOutputs` array items to include optional `slotId?: string`.

Updated outputData structure:
```typescript
const outputData = {
  fields: formData,                              // existing: by UUID

  files: fileOutputs.map(...),                   // existing: unchanged
  fileSlots: buildFileSlots(fileOutputs, formFieldDefs),   // NEW
  text: combinedText,                            // existing: unchanged

```

Create helper functions `buildFieldsByKey` and `buildFileSlots` within the same file.

**Add loadNodeConfig() helper:**

Create a private `loadNodeConfig()` function that loads the ExportConfig from the workflow definition stored in the database:

```typescript
async function loadNodeConfig(documentId: string, nodeExecutionId: string): Promise<ExportConfig | null>
```

Implementation:
1. Query `nodeExecutions` to get the current execution's `nodeId`
2. Query `documents` to get `workflowId`
3. Query `workflows` to get the workflow definition (which contains `nodes` JSONB)
4. Find the matching node by `nodeId` in the workflow's nodes array
5. Return `node.config` cast as `ExportConfig`, or null if not found

Import `documents`, `workflows` from schema (add to existing db imports).

**Update resolveContent() signature and logic:**

Change `resolveContent` to accept contentMapping:

```typescript
async function resolveContent(
  documentId: string,
  nodeExecutionId: string,
  contentMapping?: VariableRef[],
): Promise<string>
```

At the top of the function, if `contentMapping` has entries:
1. Query all nodeExecutions for this documentId (already done)
2. Map executions to `{ nodeId, outputData }` format
3. For each ref in contentMapping, call `resolveRef(ref, nodeExecs)`
4. Collect resolved values, skip nulls with `console.warn` (per user decision: skip failed refs, log warning)
5. Join with `"\n\n"` (per user decision: double newline, no titles/dividers)
6. Return joined string

If contentMapping is empty or undefined, fall through to existing upstream-scan logic (backward compatible, per user decision).
